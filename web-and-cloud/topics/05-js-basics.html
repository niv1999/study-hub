0. מהי JavaScript?

JavaScript היא שפת תכנות שתוכננה במקור כ-scripting language שרץ בתוך דפדפן. בשלושת השכבות של דף אינטרנט לכל אחת תפקיד: HTML אחראית על התוכן והמבנה, CSS על העיצוב (presentation), ו-JavaScript על ההתנהגות של האלמנטים. רק JS היא שפת תכנות אמיתית מבין השלוש.

  • ⚠️ JavaScript היא לא Java — אין ביניהן קשר, וזה לא קיצור של השם.
  • השפה כתובה בתקן ECMAScript-262; הגרסה שנלמדה היא ES2023 (הידועה גם כ-ES14).
  • זו שפה מפורשת (interpreted) — הדפדפן קורא את הקוד כטקסט מלמעלה למטה ומריץ אותו ללא קומפילציה, בדיוק כמו HTML ו-CSS.
  • ⚠️ JS אינה שפה מבוססת-עצמים קלאסית: ההורשה נעשית ב-prototyping ולא ב-Classes, ופונקציה היא גם אובייקט — "everything is an object".

מקור: 5.1_Lec_Javascript_Intro_1 (עמ' 6–7, 14–16)

1. שילוב JavaScript בדף

הדפדפן מציג כטקסט כל תו שאינו בתוך תגית. כדי לדווח לו על מעבר מ-HTML ל-JavaScript משתמשים בתגית <script>. יש שתי דרכים:

  • קוד פנימי (inline): כותבים את הקוד בין תגיות ה-<script>.
  • קובץ חיצוני: כותבים את הקוד בקובץ בסיומת .js ומקשרים אותו דרך ה-attribute בשם src. ⚠️ בקורס שלנו קובצי ה-JS נשמרים בתיקיית js.
<!-- inline -->
<script>
  console.log("Hello World!");
</script>

<!-- external file -->
<script src="js/scripts.js"></script>

1.1 מיקום התגית — למה <script> ב-head יכול "להתפוצץ"

השקפים מציבים את ה-<script> ב-head, אבל בתרגול הקוד מוצב דווקא בסוף ה-<body>. הסיבה קריטית: הדפדפן קורא את הדף מלמעלה למטה. סקריפט רגיל ב-head רץ מיד, לפני שאלמנטי ה-body נוצרו — ולכן פנייה אליהם תיכשל: 

<head>
  <script>
    // רץ לפני שה-body נבנה
    const t = document.getElementById("title");
    console.log(t);   // null !
  </script>
</head>
<body>
  <h1 id="title">שלום</h1>
</body>

שתי פתרונות נפוצים: (1) להציב את הסקריפט בסוף ה-body (כמו בתרגול), כך שכל ה-DOM כבר קיים; או (2) להשתמש ב-attribute defer על סקריפט חיצוני.

העשרה — defer מול async: שניהם חלים רק על סקריפט חיצוני (עם src) ומורידים אותו במקביל לפענוח ה-HTML:

  • defer — ההרצה נדחית עד שכל ה-HTML פוענח, ולפי סדר הופעת הסקריפטים. זהו הבחירה הבטוחה כשהקוד ניגש ל-DOM.
  • async — הסקריפט רץ ברגע שהסתיימה ההורדה, ללא הבטחת סדר, ועלול לרוץ לפני שה-DOM מוכן.
  • ללא אף אחד מהם — ההורדה וההרצה עוצרות את פענוח ה-HTML (blocking).

מקור: 5.1_Lec_Javascript_Intro_1 (עמ' 9–12), 5.2_Tar_Javascript_Intro (עמ' 2) · defer/async — העשרה

2. פלט ואינטראקציה — console, alert, prompt

לאובייקט console ארבע דרגות הדפסה, כל אחת בעיצוב משלה ב-DevTools: 

console.log("log");     // רגיל
console.error("error"); // אדום עם X
console.warn("warn");   // צהוב עם משולש אזהרה
console.debug("debug"); // כחול
  • המילה השמורה debugger; בקוד עוצרת את ריצת הדפדפן בשורה זו ("Paused in debugger") — הקוד שאחריה לא רץ עד resume.
  • alert(msg) — מציג תיבת דו-שיח חוסמת. ⚠️ המרצה מדגיש: alert לצורך דיבאג אינו מקובל — לדיבאג משתמשים ב-console.
  • prompt(msg) — מבקש קלט מהמשתמש. ⚠️ הוא תמיד מחזיר מחרוזת (או null אם ביטלו) — כדי לחשב עליו מספרית חייבים המרה עם Number().

מלכודת: const age = prompt("גיל?"); console.log(age + 1); — אם המשתמש הקליד 5, הפלט הוא "51" ולא 6, כי age הוא המחרוזת "5" ו-+ משרשר מחרוזות. הפתרון: Number(age) + 1.

מקור: 5.1_Lec_Javascript_Intro_1 (עמ' 11), 5.2_Tar_Javascript_Intro (עמ' 4–6) · alert/prompt — העשרה

3. משתנים — var, let, const

let מגדיר משתנה שניתן לשנות את ערכו, const מגדיר קבוע שלא ניתן להשמה מחדש, ו-var היא הדרך הישנה. ⚠️ המרצה מנחה מפורשות: בקורס לא משתמשים ב-var. שלושה הבדלים לזכור:

varletconst
סקופפונקציה / גלובליבלוק { }בלוק { }
השמה מחדשמותרתמותרת❌ אסורה (TypeError)
הצהרה כפולהמותרת❌ אסורה❌ אסורה

סקופ הבלוק אומר שמשתנה let/const "חי" רק בתוך הסוגריים המסולסלים שבהם הוגדר; var מתעלם מגבולות הבלוק ונשאר גלובלי: 

var a = 10;
let b = 20;
if (a) {
  var a = 4;   // עדיין הגלובלי
  let b = 6;   // מקומי לבלוק בלבד
}
console.log(a, b);   // 4 20

⚠️ השמה מחדש ל-const זורקת שגיאה: Uncaught TypeError: Assignment to constant variable; ל-let זה מותר.

3.1 Hoisting ו-TDZ

לפני ההרצה JS "מניפה" (hoisting) את כל ההצהרות לראש הסקופ — אך לא את ההשמות.

  • var מונף ומאותחל ל-undefined — שימוש לפני ההצהרה מחזיר undefined (לא שגיאה).
  • let/const מונפים אך לא מאותחלים — שימוש לפניהם זורק ReferenceError. פרק ה"אי-אתחול" הזה נקרא Temporal Dead Zone (TDZ).
  • הצהרת פונקציה (function declaration) מונפת במלואה — אפשר לקרוא לה עוד לפני שהוגדרה בקוד.
console.log(x);   // undefined (var מונף ומאותחל)
var x = 1;

console.log(y);   // ReferenceError (let ב-TDZ)
let y = 1;

מקור: 5.1_Lec_Javascript_Intro_1 (עמ' 17–18), 5.2_Tar_Javascript_Intro (עמ' 9–22)

4. טיפוסי נתונים ו-typeof

ל-JS ששה טיפוסים בסיסיים שכדאי להכיר (חמישה primitives + object):

5. אופרטורים, == מול ===, ו-truthy/falsy

אופרטורים עיקריים:

  • חשבון: + - * / % (שארית) ו-** (חזקה).
  • השמה: = += -= *=, וכן ++ / --.
  • השוואה: == === != !== < > <= >=.
  • לוגיים: && (וגם), || (או), ! (שלילה).

5.1 == מול === — הגותצ'ה הכי נבחנת

  • === (strict) — משווה ערך וגם טיפוס, ללא המרה.
  • == (loose) — קודם ממיר טיפוסים (type coercion) ואז משווה — מקור לתקלות. הכלל המעשי: תמיד להעדיף ===.
0 == "0"      // true   ("0" הומר ל-0)
0 === "0"     // false  (number מול string)
0 == ""       // true   (שניהם הומרו ל-0)
null == undefined  // true
null === undefined // false
"5" + 3   // "53"  (+ משרשר: 3 הומר למחרוזת)
"5" - 3   // 2     (- מכריח מספר: "5" הומר ל-5)

5.2 Truthy / Falsy

בתנאי (if, &&, ||) כל ערך מומר אוטומטית ל-boolean. יש בדיוק שבעה ערכים falsy — כל השאר truthy: 

false, 0, "" (מחרוזת ריקה), null, undefined, NaN, 0n

מלכודת: "0" (מחרוזת עם התו אפס) הוא truthy — כי כל מחרוזת לא ריקה היא truthy, גם "false" ו-" " (רווח). רק המחרוזת הריקה "" היא falsy. 

if ("0") console.log("A");
if (0)   console.log("B");
if ([])  console.log("C");

הפלט: A ו-C. "0" מחרוזת לא ריקה → truthy (A); 0 → falsy (B לא מודפס); [] מערך ריק הוא object ולכן truthy (C).

מקור: 5.2_Tar_Javascript_Intro (עמ' 4, 9) · אופרטורים, ==/===, truthy/falsy — העשרה

6. Template Literals והמרות טיפוסים

Template literals נכתבים בגרשיים אחוריים (backticks — התו `) במקום מרכאות. הם מאפשרים הטמעת ביטויים בתוך ${ } ומחרוזות רב-שורתיות — במקום שרשור עם +: 

const name = "Dana", n = 3;
// שרשור ישן:
"Hi " + name + ", you have " + n + " msgs"
// template literal:
`Hi ${name}, you have ${n} msgs`
`sum = ${n + 1}`   // הביטוי מחושב: "sum = 4"

6.1 המרות טיפוסים מפורשות

  • Number("42") → 42; אבל Number("12px") → NaN (המחרוזת כולה חייבת להיות מספר).
  • String(42) → "42".
  • parseInt("12px") → 12 (קורא ספרות מובילות ומתעלם מהשאר); parseInt("px12") → NaN.
  • parseFloat("3.14m") → 3.14.

לזכור: Number() "קפדן" — פסולת אחת ומקבלים NaN; parseInt() "סלחן" — קורא כמה שאפשר מההתחלה. לכן Number("12px") ≠ parseInt("12px").

מקור: 5.1_Lec_Javascript_Intro_1, 5.2_Tar_Javascript_Intro · template literals והמרות טיפוסים — העשרה

7. בקרת זרימה — תנאים ולולאות

7.1 if / else ו-switch

if (score >= 90) {
  grade = "A";
} else if (score >= 80) {
  grade = "B";
} else {
  grade = "C";
}

switch (day) {
  case 6:
  case 7:
    console.log("weekend");
    break;      // ⚠️ בלי break "נופלים" ל-case הבא
  default:
    console.log("weekday");
}

⚠️ שתי נקודות ב-switch: ההשוואה היא === (ללא המרה), ובלי break הביצוע "נופל" (fall-through) ל-case הבא.

7.2 לולאות — for, while, for..of

for (let i = 0; i < 3; i++) {
  console.log(i);        // 0 1 2
}

let i = 0;
while (i < 3) {
  console.log(i);
  i++;
}

const fruits = ["apple", "banana"];
for (const f of fruits) {  // עובר על ה-ערכים
  console.log(f);          // apple banana
}

העשרה — for..of מול for..in: for..of עובר על הערכים של מבנה ניתן-לאיטרציה (מערך, מחרוזת); for..in עובר על המפתחות/אינדקסים. על מערך, for..of ייתן "apple","banana" ואילו for..in ייתן "0","1" (מחרוזות!).

מקור: 5.1_Lec_Javascript_Intro_1, 5.2_Tar_Javascript_Intro · תנאים ולולאות — העשרה

8. בדיקה עצמית

מה יודפס לקונסול?
console.log(0 == "0", 0 === "0");

מה יודפס?
var a = 10; let b = 20; if (a) { var a = 4; let b = 6; } console.log(a, b);

איזה מהערכים הבאים הוא truthy (התנאי if ייכנס)?

מה יקרה?
console.log(greeting); let greeting = "hi";

מה יודפס?
console.log(Number("12px"), parseInt("12px"));
