0. איך לקרוא את העמוד הזה

המבחן של המרצה הוא 30 שאלות אמריקאיות ברוח המבחן לדוגמה: קצר, מעשי, ומלא "פינות חדות". זה אינו עמוד תיאוריה — זהו לקט מרוכז של המלכודות שהמרצה אוהב לבחון עליהן, מסודר לפי קטגוריות (HTML · CSS · JS · DOM · רשת · Node/Express · כלים). לכל מלכודת יש אזהרה קצרה ודוגמת קוד. אם אתם קוראים דבר אחד לפני המבחן — שיהיה זה.

שלוש שאלות שהמרצה חוזר עליהן: "מה יודפס?" (קריאת קוד ו-type coercion), "איפה הבאג?" (סדר טעינה, סוגריים, res כפול), ו"איזו הצהרה שגויה?". כמעט כל טעות בעמוד הזה היא ישירות מהשקפים; הרחבות מעבר לשקפים מסומנות ב-העשרה.

מקור: קונספט צולב מכל הרצאות הקורס (CONTRACT — עמוד מלכודות)

1. מלכודות HTML

1.1 <script> משתמש ב-src, לא ב-href

קישור (<a>) ו-<link> משתמשים ב-href; אבל <script> ו-<img> טוענים קובץ עם src. החלפה ביניהם היא מלכודת קלאסית — הסקריפט פשוט לא ייטען, בלי שגיאה רועשת. 

<!-- ✗ שגוי: הסקריפט לא ייטען -->
<script href="js/app.js"></script>

<!-- ✓ נכון -->
<script src="js/app.js"></script>

1.2 alt הוא מאפיין חובה ב-<img>

לפי השקפים alt הוא מאפיין חובה בכל שימוש בתגית התמונה: הוא מוצג כשהתמונה לא נטענת ונקרא ע"י מנועי חיפוש ו-screen readers. אל תבלבלו בין alt (טקסט חלופי, חובה) לבין title (מוצג ב-tooltip ב-hover, לא חובה). 

<img src="cat.jpg" alt="חתול ג'ינג'י על ספה">   <!-- ✓ -->
<img src="cat.jpg">                             <!-- ✗ חסר alt -->

1.3 required — אימות טופס לפני שליחה

המאפיין required על שדה <input> מונע שליחת הטופס אם השדה ריק — אימות בסיסי בצד הלקוח, בלי שורת JS אחת. שימו לב: זהו מאפיין בוליאני (נכתב לבד, בלי ערך). 

<form>
  <input type="email" name="email" required>
  <input type="submit" value="שליחה">
</form>

עוד מלכודות HTML מהשקפים: ל-<!DOCTYPE html> אין תגית סגירה והיא אינה תגית אלא הצהרה; קינון תגיות כמו סוגריים במתמטיקה (<i><b>…</b></i> תקין, סגירה מוצלבת שגויה); אין לשנות גדלי תמונות ב-HTML (רק בכלי עריכה); לוגו וכפתור-תמונה מוגדרים רק ב-CSS ולעולם לא בתגית <img>.

מקור: 1.2_Lec_HTML_Intro, 1.3_Lec_HTML_Tags (עמ' 42, 45, 48), 8.1_Lec_Forms

2. מלכודות CSS

2.1 חישוב רוחב הקופסה — width הוא רק התוכן

מהשקפים: הגדרות width/height מתייחסות לאזור התוכן בלבד. padding, border ו-margin מתווספים מעל ה-width ומגדילים את הקופסה בפועל. זו שאלת "כמה פיקסלים ייתפסו על המסך?" שחוזרת שוב ושוב. 

section {
  width: 200px;
  padding: 20px;          /* 20 מכל צד */
  border: 5px solid #000; /* 5 מכל צד */
}
/* רוחב נראה = 200 + 20+20 + 5+5 = 250px
   (ה-margin מוסיף מרווח חיצוני, לא לרוחב הקופסה עצמה) */

העשרה — box-sizing: אם נוסיף box-sizing: border-box, המשמעות של width מתהפכת: הוא כולל את ה-padding וה-border, כך שהרוחב הנראה נשאר 200px והתוכן מצטמצם. ברירת המחדל היא content-box (החישוב למעלה). border-box נוח יותר ומעבר לשקפים — אבל שווה להכיר.

2.2 ספציפיות ו-cascade — מי מנצח?

כשכמה חוקים צובעים את אותו אלמנט, לא תמיד "האחרון מנצח". קודם משווים ספציפיות: id (#) חזק מ-class (.), שחזק מסלקטור תגית. רק כשהספציפיות שווה — החוק המאוחר יותר (בסדר ה-cascade) גובר. 

p        { color: green; }  /* ספציפיות 0,0,1 */
.card p  { color: blue;  }  /* ספציפיות 0,1,1 */
#box     { color: red;   }  /* ספציפיות 1,0,0  ← מנצח */

<div class="card"><p id="box">איזה צבע?</p></div>
/* התוצאה: אדום — ה-id גובר, למרות ש-p ירוק מוגדר... אחריו? לא. לפניו. */

עוד מלכודות CSS מהשקפים: ב-margin/padding shorthand מספר הערכים והסדר משמעותיים (4 ערכים = top→right→bottom→left, בכיוון השעון); לאלמנט inline (כמו <a> או <span>) אי אפשר להגדיר width/height — צריך display: block; הסלקטור > = ילד ישיר בלבד, ו-input[type="text"] לא תופס type="password".

מקור: 3.2_Lec_CSS_Properties (עמ' 18, 24–30), 4.1_Lec_Layouts_Box_Model (עמ' 28–31)

3. מלכודות JavaScript — יסודות

3.1 == מול === ו-type coercion

== בודק ערך בלבד תוך המרת סוגים (coercion); === בודק ערך וגם סוג. בקורס מעדיפים תמיד === כדי להימנע מהפתעות. 

5 == "5"    // true   (ממיר "5" ל-5 ואז משווה)
5 === "5"   // false  (number מול string — סוג שונה)
0 == false  // true
0 === false // false

3.2 + עם מחרוזת — חישוב משמאל לימין

כשמחברים מספר ומחרוזת, JS מתייחס למספר כאל מחרוזת ומשרשר. אבל החישוב הוא משמאל לימין, ולכן הסדר קובע את התוצאה: 

16 + 4 + "Volvo"   // "20Volvo"   (16+4=20 ואז +"Volvo")
"Volvo" + 16 + 4   // "Volvo164"  (מיד הופך למחרוזת)
"5" - 2            // 3           (‏- מכריח המרה למספר!)

3.3 Template literals — backtick, לא גרש

שרשור משתנים עם ${...} עובד רק בתוך backtick (` — מקש הטילדה), לא בגרש רגיל או במרכאות. גרש רגיל יַדפיס את הטקסט כמו שהוא, בלי החלפה. 

let name = "דן";
`שלום ${name}`   // "שלום דן"      ✓ backtick
'שלום ${name}'   // "שלום ${name}"  ✗ מחרוזת רגילה — אין החלפה

3.4 Hoisting, TDZ ו-var/let/const

רק הצהרות מונפות, לא השמות. var מונף ומאותחל ל-undefined; let/const מונפים אך לא מאותחלים (אזור ה-TDZ), ולכן שימוש לפני ההצהרה זורק ReferenceError. בקורס לא משתמשים ב-var. 

console.log(a);  // undefined         (var מונף ומאותחל)
var a = 5;

console.log(b);  // ReferenceError    (let ב-TDZ)
let b = 5;

const c;         // SyntaxError: Missing initializer  (const חייב ערך מיד)

const נועל את ההשמה, לא את התוכן: const arr = [1]; arr.push(2); מותר (משנים תוכן), אבל arr = [3]; זורק TypeError: Assignment to constant variable. אותו דבר לאובייקט: מותר לשנות property, אסור להשים מחדש.

מקור: 5.1_Lec_Javascript_Intro (עמ' 17–18), 6.1_Lec_Javascript_Cont (עמ' 7–12, 23, 55)

4. מלכודות JavaScript — פונקציות, מערכים ו-this

4.1 Closure בלולאה עם var

המלכודת המפורסמת ביותר: var משותף לכל האיטרציות (function-scoped), ולכן כל ה-callbacks האסינכרוניים רואים את אותו i — הערך שלו אחרי שהלולאה הסתיימה. עם let (block-scoped) נוצר עותק חדש בכל סבב. 

for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);
}
// מדפיס: 3 3 3   (i אחד משותף, ערכו 3 בסיום הלולאה)

for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);
}
// מדפיס: 0 1 2   (let — עותק חדש לכל איטרציה)

4.2 this בפונקציית חץ מול פונקציה רגילה

לפונקציית חץ אין this משלה — היא יורשת את ה-this מהסקופ החיצוני (inner this = outer this). פונקציה רגילה בתוך setTimeout "מאבדת" את ה-this ומחזירה undefined — בדיוק הדוגמה מהשקפים. 

this.speed = 50;
setTimeout(function () { console.log(this.speed); }, 1000); // undefined
setTimeout(() =>         console.log(this.speed),  1000);   // 50  ✓

4.3 מערכים: map מחזיר חדש, forEach לא

map יוצר ומחזיר מערך חדש; forEach רק רץ על האיברים ומחזיר undefined. השמה מ-forEach למשתנה היא באג נפוץ. 

const a = [1, 2, 3];
const b = a.map(x => x * 2);     // b = [2,4,6],  a לא השתנה
const c = a.forEach(x => x * 2); // c = undefined  (לא מחזיר מערך!)

4.4 by-value מול by-reference

פרימיטיבים (number, string, boolean) מועברים לפונקציה כהעתק (by-value); אובייקטים ומערכים מועברים ברפרנס — שינוי בפונקציה משנה את המקור. 

function f(n, o) { n = 99; o.x = 99; }
let num = 1; let obj = { x: 1 };
f(num, obj);
console.log(num); // 1   (פרימיטיב — העתק)
console.log(obj); // { x: 99 }  (אובייקט — רפרנס)

העשרה — typeof: typeof [] מחזיר 'object' ולא 'array'; typeof null מחזיר 'object' (באג היסטורי מפורסם); typeof NaN הוא דווקא 'number'.

מקור: 6.1_Lec_Javascript_Cont (עמ' 4–5, 60–61), 5.1_Lec_Javascript_Intro (עמ' 18) + העשרה

5. מלכודות DOM ואירועים

5.1 <script> ב-head רץ לפני שה-DOM קיים

הדפדפן קורא מלמעלה למטה. סקריפט ב-head שמנסה לגשת לאלמנט ב-body ירוץ לפני שהאלמנט קיים, ו-getElementById יחזיר null → TypeError. 

<head>
  <script>
    // ✗ הדפדפן עדיין לא קרא את ה-body:
    document.getElementById("link").innerHTML;
    // Uncaught TypeError: Cannot read properties of null
  </script>
</head>
<body><a id="link">Click here</a></body>

שלושה פתרונות: (1) לשים את ה-<script> בסוף ה-body (אחרי האלמנטים); (2) לעטוף ב-window.onload — window.onload = () => {...} (נורה אחרי טעינת כל המשאבים); (3) העשרה: להוסיף defer ל-script ב-head (<script src="app.js" defer>) — הדפדפן ידחה את ההרצה עד שה-DOM מוכן.

5.2 addEventListener — שם הפונקציה בלי סוגריים

מעבירים ל-addEventListener (או ל-onclick) את שם הפונקציה, בלי (). אם תוסיפו סוגריים — הפונקציה תרוץ מיד בזמן הרישום, וערך ההחזרה שלה (בד"כ undefined) יירשם כ-handler. 

btn.addEventListener("click", doStuff);    // ✓ ירוץ בלחיצה
btn.addEventListener("click", doStuff());  // ✗ רץ מיד, לא בלחיצה
btn.onclick = doStuff;                      // ✓
btn.onclick = doStuff();                    // ✗

חיבור handler ישירות ב-HTML (onclick="doIt()") הוא "שיטה ישנה" וערבוב שכבות — בקורס מחברים אירועים בקובץ JS חיצוני, בתוך window.onload.

מקור: 6.1_Lec_Javascript_Cont (עמ' 25–28, 41–46) + העשרה (defer)

6. מלכודות רשת — fetch, JSON ו-HTTP

6.1 fetch לא "נכשל" על 404 או 500

זו הפינה החדה הכי מפורסמת: fetch מקיים (resolve) את ה-Promise גם כשהשרת מחזיר 404 או 500. ה-Promise נדחה (reject) רק על כשל רשת (אין חיבור). לכן חייבים לבדוק בעצמכם את response.ok או response.status. 

fetch("/api/missing")            // השרת מחזיר 404
  .then(res => {
    // מגיעים לכאן גם ב-404!  res.ok === false
    if (!res.ok) throw new Error(res.status);
    return res.json();
  })
  .catch(err => console.log("שגיאה", err));

6.2 POST של JSON — JSON.stringify + Content-Type

כששולחים אובייקט ב-POST, חייבים שלושה דברים: מתודה POST, כותרת Content-Type: application/json, וגוף שהומר למחרוזת עם JSON.stringify (אי אפשר לשלוח אובייקט JS גולמי). 

fetch("http://127.0.0.1:8081/api/posts", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ title: "Hi", description: "..." })  // חובה stringify
});
// זכרו: JSON.parse = מחרוזת → אובייקט ,  JSON.stringify = אובייקט → מחרוזת

6.3 קודי סטטוס — 400 לקלט לא תקין

משפחות: 2xx הצלחה, 3xx הפניה, 4xx שגיאת לקוח, 5xx שגיאת שרת. הכי נבחנים:

  • 200 OK — הצלחה.
  • 400 Bad Request — הקלט מהלקוח לא תקין (שדה חסר/פורמט שגוי).
  • 404 Not Found — המשאב לא נמצא.
  • 500 Internal Server Error — קרס משהו בשרת.

6.4 CORS — פורט שונה = origin שונה

origin = פרוטוקול + host + פורט. אם הלקוח רץ על localhost:5500 וה-API על localhost:8081 — אלה שני origins שונים (רק בגלל הפורט!), והדפדפן יחסום את הבקשה אלא אם השרת שולח כותרת Access-Control-Allow-Origin. בשקפי הקורס השרת מגדיר Access-Control-Allow-Origin: * בדיוק בשביל זה.

מקור: 7.1_Lec_Client_Server (עמ' 20, 28), 12.1_Lec_Full-Stack (עמ' 21, 24) + העשרה (fetch .ok, CORS origin, 400)

7. מלכודות Node ו-Express

7.1 חובה לשלוח בדיוק res אחת

לכל בקשה יש תגובת HTTP אחת. שליחת שתי תגובות (או res.send אחרי res.json) זורקת שגיאה. מהשקפים: אחרי res.send() הקוד בצד השרת לא ממשיך — אבל אם תגיעו בטעות ל-send שני, תקבלו את השגיאה. 

app.get("/x", (req, res) => {
  res.send("first");
  res.send("second");  // ✗ Error: Cannot set headers after they are sent
});
// הלקוח יקבל "first"; ה-"second" יזרוק שגיאה בשרת.

מלכודת נלווית: if/else ששני הענפים שלו שולחים res — תקין. אבל res בתוך if ועוד אחד אחרי הלולאה בלי return — עלול לשלוח פעמיים. שימו return אחרי res.send כשצריך לעצור.

7.2 express.json() — בלעדיו req.body הוא undefined

ברירת המחדל: req.body הוא undefined. כדי שאקספרס יפרסר גוף JSON חייבים את ה-middleware app.use(express.json()) (ו-express.urlencoded() לטפסים). המודול body-parser מובנה מגרסה 4 — לא להשתמש בו ישירות. 

const app = express();
app.use(express.json());               // בלי זה — req.body === undefined
app.use(express.urlencoded({ extended: true }));

app.post("/users", (req, res) => {
  console.log(req.body.username);      // עובד רק בזכות express.json()
  res.status(200).json({ username: req.body.username });
});

7.3 :id (path param) מול ?query

שני מנגנונים שונים לשלוח פרמטרים: path param הוא חלק מהנתיב (/user/:id) ונקרא מ-req.params; query string בא אחרי ? ונקרא מ-req.query. אל תערבבו. 

app.get("/user/:id", (req, res) => res.send(req.params.id));
// GET /user/5           → "5"

app.get("/find", (req, res) => res.send(req.query.author));
// GET /find?author=King → "King"
// GET /find (בלי query) → undefined  (חסר = undefined, לא שגיאה)

סדר ה-routes חשוב: כל route הוא כמו if/else — נכנסים רק לראשון שמתאים. אם app.all('*') מוגדר לפני app.get('/'), הוא יתפוס הכול. כדי להמשיך בשרשרת צריך פרמטר שלישי next וקריאה ל-next().

מקור: 10.1_Lec_Express (עמ' 13–14, 18–22, 29), 12.1_Lec_Full-Stack (עמ' 10)

8. מלכודות כלים — Git, npm ו-.env

8.1 .env חייב להיות ב-.gitignore

קובץ .env מחזיק סודות (מפתחות API, connection string ל-DB, סיסמאות). אסור להעלות אותו ל-Git — מוסיפים אותו ל-.gitignore. אותו דבר ל-node_modules/: הוא כבד ומשוחזר מ-package.json, אז לא מעלים אותו לריפו. 

# .gitignore
node_modules/

8.2 אחרי clone — מריצים npm install

כשמשכפלים (clone) פרויקט Node, תיקיית node_modules לא מגיעה איתו (היא ב-.gitignore). לפני שמריצים משהו חייבים npm install — הוא קורא את package.json ומוריד את כל התלויות מחדש. 

git clone https://github.com/user/repo.git
cd repo
npm install     # מוריד את node_modules לפי package.json
node index.js   # רק עכשיו אפשר להריץ

העשרה — deploy: אותו היגיון תקף בפריסה ל-Render — משתני הסביבה מ-.env מוגדרים בממשק כ-Environment Variables (לא מעלים את הקובץ), והפלטפורמה מריצה npm install אוטומטית בכל דחיפה.
