0. מה זה CSS ולמה מפרידים שכבות

CSS ראשי תיבות של Cascading Style Sheets — "גיליונות סגנון מדורגים". זו אינה שפת תכנות אלא רשימה של הגדרות עיצוב שקובעות כיצד יוצגו האלמנטים של מסמך ה-HTML. אם HTML אחראי על התוכן והמבנה (content), הרי ש-CSS אחראי על אופן התצוגה (presentation).

הפרדת שכבות (Separation of Concerns): אפשר להחליף לגמרי את העיצוב של אתר על ידי שינוי קובץ ה-CSS בלבד — בלי לגעת בקוד ה-HTML. כלל אצבע ויזואלי מהתרגול: סוגריים משולשים < > פירושם HTML, וסוגריים מסולסלים { } פירושם CSS.

מקור: 3.1_Lec_CSS_Intro_1 (עמ' 4, 7)

1. שלוש דרכים לשלב CSS — ומי המומלצת

אפשר לצרף CSS לדף HTML בשלוש דרכים:

1.1 Inline — עיצוב על התגית עצמה

הגדרת העיצוב נכתבת כמאפיין style על התגית עצמה, כאשר מפרידים בין הגדרה להגדרה בנקודה-פסיק ;: 

<b style="background-color:gold; color:blueviolet; font-size:24px;">Text</b>

הבעיות: אין הפרדת שכבות, תחזוקה קשה (מתקנים כל אלמנט בנפרד), אין שימוש חוזר, ואין מדרגיות.

1.2 Internal — תגית <style> בתוך ה-<head>

פותר חלק מהבעיות: אפשר לעצב את כל האלמנטים מאותו סוג במקום אחד באותו עמוד.

<head>
    <style>
        b { background-color: gold; color: blueviolet; font-size: 24px; }
    </style>
</head>

1.3 External — קובץ .css נפרד

כותבים את כל הגדרות העיצוב בקובץ נפרד ומקשרים אליו מתוך ה-<head> עם תגית <link>. ה-href מציין את מיקום ושם קובץ העיצוב: 

<head>
    <link rel="stylesheet" href="css/style.css">
    <title>External CSS file</title>
</head>

בקורס עובדים אך ורק בשיטת External! יתרונותיה: הפרדת שכבות מלאה, תחזוקה במקום מרכזי אחד, שימוש חוזר (אותו עיצוב על אובייקטים רבים), ומדרגיות (קובץ CSS אחד יכול לשרת דפי HTML רבים). בשקפים לפעמים מוצג Inline/Internal כי כך נוח להדגים — אבל לא כך מתכנתים.

מלכודת נתיבים: קובץ ה-CSS יושב בדרך כלל בתיקיית css/ נפרדת. לכן נתיב לתמונה מתוך ה-CSS חייב "לעלות תיקייה": url("../images/star.png") — ה-../ יחסי למיקום קובץ ה-CSS, לא למיקום ה-HTML.

מקור: 3.1_Lec_CSS_Intro_1 (עמ' 8–13), 3.3_Tar_CSS_Intro (עמ' 3, 6)

2. תחביר ה-Rule וסוגי הסלקטורים

כל הגדרת עיצוב (rule) בנויה מסלקטור וגוף בסוגריים מסולסלים, שבתוכו זוגות של מאפיין: ערך:

selector {
    property: value;   /* חובה נקודה-פסיק בסוף כל הגדרה */
}

ה-selector קובע על מי חל העיצוב. הערות ב-CSS נכתבות רק בפורמט /* ... */ (אין //). להלן סוגי הסלקטורים:

סלקטורדוגמהעל מי חל
תגית (element)p { }כל תגיות ה-p בעמוד
class (נקודה).intro { }כל אלמנט עם class="intro" — ניתן לחזרה
id (סולמית)#logo { }אלמנט יחיד עם id="logo"
קיבוץ (group)p, h1, h2 { }אותו עיצוב לכל הסלקטורים ברשימה
צאצאים (descendant)nav a { }כל a שאיפשהו בתוך nav
אוניברסלי (כוכבית)* { }כל האלמנטים בעמוד

id מול class — הכללים שנבחנים: id חייב להופיע פעם אחת בלבד בעמוד, נכתב ב-CSS עם # וללא רווחים, ולא ניתן לתת שני id-ים לאותו אלמנט. class נכתב עם ., יכול לחזור על אלמנטים רבים, וניתן לתת לאלמנט שני קלאסים עם רווח ביניהם: class="blue background".

הרווח משנה משמעות! p.blue (בלי רווח) = תגית p שגם בעלת class="blue" (אותו אלמנט). nav a (עם רווח) = סלקטור צאצא — כל a שנמצא בתוך nav. ואילו > בוחר ילד ישיר בלבד: #main-nav > li תופס רק li שהם ילדי-דור-ראשון של #main-nav, לא נכדים.

העשרה — סלקטורים נוספים מהשקפים: h1 + p = אח עוקב מיידי (רק הפסקה הראשונה שאחרי h1); h2 ~ p = כל אח עוקב; [title] = כל אלמנט עם המאפיין title; input[type="text"] = לפי ערך מדויק (שדה type="password" לא נתפס). ה-* משמש בעיקר לאיפוס: * { margin: 0; padding: 0; } מוריד את השוליים הדיפולטיים של הדפדפן.

מקור: 3.1_Lec_CSS_Intro_1 (עמ' 22–35, 41), 3.2_Lec_CSS_Properties_1 (עמ' 24–30)

3. ספציפיות, ה-Cascade וירושה — מי גובר?

ניתן להגדיר עיצוב לאותו אלמנט בכמה מקומות. כשאין סתירה — כל ההגדרות חלות יחד. כשיש סתירה על אותו מאפיין, מכריעים לפי היררכיה. הכלל הכללי: "מבפנים החוצה" — מה שיותר קרוב ויותר ספציפי גובר (מספר עדיפות נמוך יותר = חזק יותר).

3.1 עדיפות לפי סוג הסלקטור

סלקטורעדיפות
id (#)1 (הכי חזק)
class (.)2
שם תגית (tag)3 (הכי חלש)

3.2 עדיפות לפי מקור ההגדרה

מקורעדיפות
Inline style1 (הכי חזק)
Internal style2
External style sheet3
Browser default4 (הכי חלש)

ספציפיות חלה רק על מאפיינים מתנגשים. אם Inline קובע רק color, אבל Internal קובע גם background-color וגם font-size — ה-Inline מנצח רק על color, ושאר ההגדרות של ה-Internal עדיין חלות. אל תזרקו את כולן!

!important דורס את ההיררכיה. מאפיין עם !important גובר גם על ספציפיות גבוהה יותר — כך class עם !important ינצח id רגיל. זה קיים לידע כללי, אבל עדיף להימנע: הוא שובר את סדר העדיפויות ומקשה על התחזוקה.

העשרה — ירושה (Inheritance): חלק מהמאפיינים עוברים בירושה מאב לצאצאים כברירת מחדל — בעיקר מאפייני טקסט כמו color, font-family, font-size. לעומתם, מאפייני "קופסה" כמו margin, padding, border ו-background אינם עוברים בירושה. לכן הגדרת color על body "מטפטפת" לכל הטקסט בעמוד, אך border על body לא.

מקור: 3.1_Lec_CSS_Intro_1 (עמ' 27–32) · ירושה — העשרה

4. צבעים ויחידות מידה

color מגדיר את צבע הפונט; background-color מגדיר את צבע הרקע. צבע ניתן להגדרה בכמה דרכים: 

color: royalblue;            /* name  — שם צבע */
color: #444444;              /* hex   — 6 ספרות הקס */
color: rgb(255, 0, 0);       /* rgb   — אדום/ירוק/כחול 0..255 */
color: rgba(0, 255, 0, 0.8); /* rgba  — עם ערוץ שקיפות (alpha) 0..1 */

ה-alpha ב-rgba נע בין 0 (שקוף לגמרי) ל-1 (אטום). למשל rgba(0,255,0,0.8) הוא ירוק ב-80% אטימות.

4.1 יחידות מידה

המקובלת ביותר היא px (פיקסלים, יחידה מוחלטת). לצדה יש יחידות יחסיות שימושיות:

  • px — פיקסלים, ערך מוחלט (font-size: 16px).
  • % — אחוז יחסית לאלמנט האב (width: 50%, margin: 5% auto).
  • em — יחסית ל-font-size של האלמנט עצמו (2em = פי 2 מהגופן הנוכחי).
  • rem — יחסית ל-font-size של שורש הדף (html), ולכן עקבי בכל העמוד.

העשרה — em מול rem: השקפים מדגימים בעיקר px ו-%, אך שווה להכיר: em "מצטבר" לפי הגופן של האב (עלול להכפיל בהיררכיה עמוקה), בעוד rem תמיד יחסי לשורש — ולכן צפוי יותר. בברירת מחדל 1rem = 16px.

מקור: 3.2_Lec_CSS_Properties_1 (עמ' 21) · em/rem — העשרה

5. תכונות טקסט, רקע ו-Border

5.1 תכונות טקסט ופונט

  • color — צבע הטקסט.
  • font-family — רשימת פונטים (web-safe): הדפדפן מנסה את הראשון, ואם אינו מותקן — עובר לבא: font-family: Arial, Verdana, sans-serif;
  • font-size / font-weight — גודל ומשקל (bold / 500).
  • text-align — יישור (left/center/right).
  • text-decoration — קו תחתי; ביטול ברירת המחדל של קישורים: a { text-decoration: none; }
  • text-transform: uppercase; — אותיות גדולות.

text-decoration הוא shorthand של -line / -color / -style / -thickness. שינוי מראה במעבר עכבר נעשה עם ה-פסאודו-מחלקה a:hover, ולקישור שביקרו בו — a:visited.

5.2 רקע ו-Border

#box {
    background-color: bisque;
    background-image: url("../images/star.png");
    background-repeat: no-repeat;   /* repeat-x / repeat-y / no-repeat */
    border: 1px solid #ffffff;      /* thickness | style | color */
}

עם no-repeat רואים את background-color בכל השטח שהתמונה לא מכסה. ה-shorthand background מאחד color/image/repeat/position — אך לא את background-size (אותו מגדירים בנפרד).

מלכודת inline מול block: לאלמנט inline (כמו <a> או <span>) אי אפשר להגדיר height/width. הפתרון: display: block;. זו הסיבה שבמימוש לוגו (תמיד קישור <a> עם תמונת רקע ב-CSS) מגדירים display: block.

מקור: 3.2_Lec_CSS_Properties_1 (עמ' 5–20, 32–35), 3.3_Tar_CSS_Intro (עמ' 22–24, 37)

6. בדיקה עצמית

נתון הקוד הבא. מה ייראה על המסך עבור המילה Text?

<style>
    b { background-color: gold; color: blueviolet; font-size: 24px; }
</style>
...
<b style="color: #ff0000">Text</b>

איזה עיצוב יחול על הפסקה?

<style>
    .paragraph { color: green; font-size: 12px; }
    #text      { color: blue;  font-size: 22px; }
</style>
...
<p class="paragraph" id="text">Lorem</p>

בהינתן הכלל p.blue { color: royalblue; } — אילו אלמנטים ייצבעו בכחול?

<h1 class="blue">A</h1>
<p  class="blue">B</p>
<p>C</p>

בהינתן header { margin: 1px auto 2em; } (שלושה ערכים) — מהו השוליים התחתונים (margin-bottom)?

מה יהיה צבע הפסקה?

.paragraph { color: green !important; }
#text      { color: blue; }
...
<p class="paragraph" id="text">Lorem</p>
