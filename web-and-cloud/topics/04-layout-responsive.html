0. display והזרימה — block, inline, inline-block, none

לכל אלמנט יש "אופי" של תצוגה. block מול inline הוא ההבדל היסודי: אלמנט block מוריד שורה ותופס אוטומטית 100% מהרוחב הזמין לו (בתוך המכיל שלו); אלמנט inline אינו מוריד שורה, ורוחבו נקבע לפי התוכן בלבד — אי אפשר להגדיר לו width או height.

  • תגיות block עיקריות: <div>, <h1>–<h6>, <p>, <ul>/<ol>/<li>. רוב שאר התגיות הן inline.
  • כלל ההכלה: block יכול להכיל גם block וגם inline; אלמנט inline יכול להכיל inline בלבד.
  • ⚠️ התגיות הסמנטיות של HTML5 (section, nav, aside, header, footer, article, main) אמורות להיות block — אך רוב הדפדפנים מציגים אותן כ-inline כברירת מחדל, ולכן יש להפוך אותן ל-block ידנית.

המאפיין display משנה את אופן ההצגה (לא את מהות האלמנט). ארבעת הערכים בקורס:

  • block — האלמנט תופס שורה משלו; אי אפשר להעמיד לידו inline.
  • inline — האלמנט נכנס לשורה, ללא שליטה על גובה ורוחב.
  • inline-block — באותה שורה כמו inline, אך עם שליטה על גובה ורוחב. ⚠️ החיסרון: נוצר רווח של כ-4 פיקסלים בין האלמנטים שפוגע בעיצוב.
  • none — מעלים את האלמנט מהמסך לגמרי; הוא אינו תופס מקום בזרימה.

המלכודת הקלאסית: נותנים ל-div ערך display: inline ומגדירים לו מידות — המידות מתעלמות. רק inline-block משמר את המידות תוך שמירה על אותה שורה: 

div { display: inline; width: 150px; height: 150px; }
/* → הרוחב/גובה מתעלמים; ה-div יוצג ברוחב התוכן בלבד */

div { display: inline-block; width: 150px; height: 150px; }
/* → ריבוע 150×150 באותה שורה (+ רווח ~4px) */

הזרימה הטבעית (normal flow): ללא הגדרה אחרת — אלמנטי block מסתדרים זה תחת זה, אלמנטי inline זה ליד זה, לפי סדר הופעתם בקוד. גובה של אלמנט inline נקבע לפי התוכן הגבוה בו או לפי line-height — והוא אינו מושפע מ-padding/border/margin אנכיים (רק שוליים אופקיים משפיעים על הרווח האופקי). זו נקודה שחוזרת במבחנים.

מקור: 4.1_Lec_Layouts_and_Box_Model_1 (עמ' 4–18, 36)

1. מודל הקופסה — Content, Padding, Border, Margin

מודל הקופסה (Box Model) קובע איך ואיפה כל אלמנט מוצג. כל אלמנט הוא קופסה בת ארבע שכבות, מבפנים החוצה:

  • Content — אזור התוכן, האלמנט עצמו.
  • Padding — ריפוד פנימי שעוטף את התוכן.
  • Border — מסגרת סביב אזור ה-Padding.
  • Margin — שוליים חיצוניים שקופים ליצירת רווח בין אלמנטים.

⚠️ שני עוגני מבחן: (א) הגדרת background צובעת את Content + Padding בלבד (ה-margin נשאר שקוף). (ב) padding מגדיל את הגודל הנראה של האלמנט ואינו יוצר רווח בין אלמנטים; רק margin יוצר רווח מחוץ לאלמנט. שלושה בלוקים צמודים עם padding בלבד ייראו כגוש אחד; עם margin ייראו כשלושה נפרדים.

1.1 חישוב הרוחב הכולל (ברירת מחדל — content-box)

זו נקודת החישוב הכי נבחנת. ⚠️ width ו-height מתייחסים לאזור התוכן בלבד — ה-padding, ה-border וה-margin מתווספים מעליו. לכן הרוחב הכולל שהאלמנט תופס בזרימה: 

רוחב כולל = width + (padding×2) + (border×2) + (margin×2)

דוגמה מההרצאה: רוצים אלמנט שסך המקום שלו הוא 200px, עם margin: 15px ו-padding: 10px מכל צד. איזה width נגדיר?

section { width: 150px; margin: 15px; padding: 10px; }
/* 150 + 10·2 + 15·2 = 200px  (אין border כאן) */

העשרה — box-sizing: content-box מול border-box. ברירת המחדל היא content-box — שם width הוא התוכן בלבד, וה-padding/border מתווספים (כמו למעלה). לעומת זאת עם box-sizing: border-box ה-width כולל בתוכו את ה-padding וה-border, כך שהמידה שהגדרת היא הגודל הסופי של הקופסה הנראית (ה-margin עדיין חיצוני). חישוב מדויק: אלמנט width: 200px; padding: 20px; border: 5px — 

content-box:  רוחב נראה = 200 + 20·2 + 5·2 = 250px , content = 200px
border-box:   רוחב נראה = 200px            , content = 200 − 40 − 10 = 150px

(דפוס נפוץ מאוד באתרים אמיתיים: * { box-sizing: border-box; }. מעבר לשקפים — להעשרה.)

2. מרכוז אופקי ואיחוד שוליים (margin collapsing)

מרכוז אופקי של אלמנט block בעל רוחב קבוע נעשה בשוליים אוטומטיים: margin: 0 auto;. בקיצור של שני ערכים, הראשון (0) הוא לשוליים האנכיים והשני (auto) לשוליים האופקיים. הדפדפן מחלק את הרוחב הנותר שווה בשווה בין שני הצדדים — וזה מה שיוצר את המרכוז. דפוס נפוץ: wrapper עוטף עם width קבוע ו-margin: 0 auto.

אל תבלבלו: text-align: center ממרכז את הטקסט בתוך האלמנט; margin: 0 auto ממרכז את האלמנט עצמו. אלה שני דברים שונים — שאלה שאוהבים לשלב.

קיצור ה-margin תלוי במספר הערכים: ערך אחד = כל הצדדים; שניים = אנכי | אופקי; ארבעה = בכיוון השעון top · right · bottom · left.

2.1 איחוד שוליים — margin collapsing

איחוד שוליים הוא מלכודת אמיתית מהתרגול: כאשר יש margin אנכי על אובייקט ריק בתוך אב ריק, השוליים על ציר ה-y "בורחים" החוצה ומאוחדים — ונוצר רווח לא-רצוי מעל האב (למשל רווח מעל header שלא הגדרנו). התיקון שהודגם: לגרום לאב לא להיות ריק — 

header {
    height: 130px;
    padding-top: 1px;   /* מבטל את איחוד השוליים */
}

⚠️ בתוך flex container אין איחוד שוליים כלל — השוליים של הפריטים אינם מתאחדים. זו אחת הסיבות ש-Flexbox מפשט layout.

מקור: 4.1_Lec_Layouts_and_Box_Model_1 (עמ' 20–26), 4.3_Tar_Box_Model_Responsive (עמ' 11–12)

3. מיקום — position ו-z-index

למקם layout שלם בעזרת margin גדולים הופך כאוטי במהירות. הכלי הנכון הוא position עם ארבעת הערכים הנלמדים בקורס:

  • static — ברירת המחדל. האלמנט בזרימה הטבעית, ואינו מושפע מ-top/right/bottom/left ולא מ-z-index.
  • relative — נזוז יחסית למקומו המקורי, אך ממשיך לתפוס את השטח המקורי בזרימה (האלמנטים האחרים לא זזים; הוא יכול לחפוף אותם).
  • absolute — יוצא מהזרימה (לא תופס מקום, השאר מתייחסים אליו כלא-קיים), וממוקם ביחס לאב הקרוב ביותר שהוגדר לו position שאינו static — ואם אין כזה, ביחס לאלמנט הראשון בדף.
  • fixed — קבוע ביחס לחלון (viewport) ואינו זז בגלילה (שימושי ל-header/footer קבועים).

⚠️ block שהופך absolute מאבד את רוחב ה-100% האוטומטי ומקבל רוחב לפי התוכן (shrink-to-fit) — בדומה ל-inline, אך כאן כן אפשר להגדיר לו רוחב ידני.

3.1 z-index — מי מעל מי

כשאובייקטים חופפים ללא z-index, זה שמופיע מאוחר יותר בקוד נראה מלמעלה. את סדר השכבות שולטים עם z-index — מספר גבוה יותר = שכבה עליונה יותר. ⚠️ z-index עובד רק על אלמנט עם position שאינו static. לכן דפוס נפוץ הוא position: relative על אלמנט כדי "להפעיל" עליו את ה-z-index: 

#div3 { position: relative; z-index: 1; }   /* יעלה מעל */
#div2 { position: absolute; z-index: -1; }  /* ירד מטה */

העשרה — position: sticky. ערך חמישי שלא הופיע בשקפים אך נפוץ מאוד: האלמנט מתנהג כ-relative עד שהגלילה מגיעה לסף שהגדרנו (למשל top: 0), ואז "נדבק" למקומו כמו fixed כל עוד ההורה גלוי. שימושי לכותרות טבלה ולסרגלי ניווט. (מעבר לחומר הקורס — להעשרה.)

מקור: 4.1_Lec_Layouts_and_Box_Model_1 (עמ' 35–54) · sticky — העשרה

4. Flexbox

Flexbox הוא מודל CSS לסידור בלוקים זה-לצד-זה ולרספונסיביות. יש בו שני תפקידים: Flex Container (האב) ו-Flex Items (הילדים הישירים). מגדירים display: flex על האב, וכל תגית בתוכו הופכת אוטומטית ל-flex item. 

.container { display: flex; }       /* מרונדר כמו block */
.container { display: inline-flex; } /* מרונדר כמו inline */

המאפיינים המרכזיים על ה-container:

  • justify-content — יישור על הציר הראשי: flex-start, flex-end, center, space-between, space-around, space-evenly. הצמדת שני אלמנטים לקצוות מנוגדים = space-between.
  • align-items — יישור על הציר המשני: flex-start, flex-end, center, stretch, baseline.
  • flex-wrap — ברירת המחדל דוחסת את הכל לשורה אחת; wrap גורם לפריטים "ליפול" לשורה חדשה כשאין מקום.
  • flex-grow — מחלק את המקום הפנוי לפי יחס הערכים (כולם 1 = חלוקה שווה; 2 = כפול מ-1). ⚠️ ברירת מחדל 0; ערכים שליליים אינם חוקיים.
section#aside-and-boxes {
    display: flex;
    justify-content: space-between;  /* aside בצד אחד, main בצד השני */
}

העשרה — flex-direction ו-gap. flex-direction קובע את כיוון הציר הראשי: row (ברירת מחדל, אופקי), column (אנכי), וכן row-reverse / column-reverse — החלפת הכיוון מחליפה בפועל בין justify-content ל-align-items. gap מגדיר רווח אחיד בין הפריטים בלי לפזר margin לכל אחד. שניהם מעבר לשקפים ומובאים להעשרה בלבד.

מקור: 4.1_Lec_Layouts_and_Box_Model_1 (עמ' 55–61), 4.3_Tar_Box_Model_Responsive (עמ' 14–24) · flex-direction/gap — העשרה

5. רספונסיביות — viewport, media queries, mobile-first

עיקרון היסוד של הקורס: אין מערכת רק לדסקטופ, אין משתמש שהוא רק על דסקטופ. עיצוב רספונסיבי מתאים את מה שהגולש רואה למסכים בגדלים שונים.

5.1 תגית ה-viewport

תגית ה-viewport נכתבת ב-<head>. בלעדיה, הדפדפן במובייל מרנדר את הדף ברוחב דסקטופ ומציג אותו מוקטן (הגולש נאלץ לזום ידנית): 

<meta name="viewport" content="width=device-width,initial-scale=1">

5.2 Media Queries

Media queries הן אבן היסוד: הן מפרידות בין טווחי רוחב שונים ומאפשרות CSS שונה לכל מדיה. בקורס כותבים אותן בתוך קובץ ה-CSS. נקודות העצירה (breakpoints) נקבעות היכן ש"העיצוב נשבר" — לא לפי רשימת מכשירים קבועה. 

@media only screen and (max-width:480px) { /* מובייל */ }
@media only screen and (min-width:481px) and (max-width:768px) { /* טאבלט */ }
@media only screen and (min-width:769px) { /* דסקטופ */ }

⚠️ ה-cascade חשוב למבחן: מה שלא נדרס בתוך ה-media query נשאר מהעיצוב הגנרי שמחוצה לו. לכן כותבים בתוך ה-media query רק את מה שמשתנה. טווח ביניים דורש שילוב שני תנאים עם and.

5.3 mobile-first, יחידות גמישות ותמונות

גישת mobile-first: מתמקדים בחשוב, מפשטים תוכן ופונקציונליות, מתעדפים תוכן וממקסמים חוויית משתמש. במקום מידות קבועות משתמשים ביחידות גמישות — בעיקר אחוזים. תמונות רספונסיביות: 

img { width: 100%; }      /* ממלא את רוחב האב (גם מותח מעבר לגודלו) */
img { max-width: 100%; }   /* עד רוחב האב בלבד — לא ימתח מעבר לגודלו */

⚠️ החיסרון של שתי השיטות: משקל התמונה — תמונה קטנה שנמתחת נראית "מרוחה", ותמונה גדולה פוגעת בביצועים, והמובייל נפגע ראשון.

העשרה — יחידות גמישות נוספות. מעבר לאחוזים, קיימות em/rem (יחסית לגודל הגופן — rem יחסי לשורש), ו-vw/vh (אחוז מרוחב/גובה ה-viewport). הן משלימות את גישת ה-mobile-first ומאפשרות עיצוב שמתכווץ חלק. (מעבר לשקפים — להעשרה.)

6. בדיקה עצמית

אלמנט מוגדר width:100px; padding:10px; border:5px solid; margin:20px (box-sizing בברירת מחדל). מהו הרוחב הכולל שהאלמנט תופס בזרימה, כולל השוליים?

בהנחה box-sizing: border-box, לאלמנט width:200px; padding:20px; border:5px solid. מהו רוחב אזור התוכן (content) בפועל?

נתון div { display: inline; width: 150px; height: 150px; } על שני div עוקבים. מה יוצג על המסך?

איזה מהמשפטים הבאים על position הוא שגוי?

בקובץ CSS: section { color: blue; font-size: 20px; } ואז @media (max-width:480px) { section { color: red; } }. ברוחב חלון של 400px — מה יהיו הצבע וגודל הגופן של section?
