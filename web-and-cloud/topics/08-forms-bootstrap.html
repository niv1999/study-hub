1. הטופס — <form>, action ו-method

טופס (form) הוא הכלי שבו המשתמש מזין מידע ושולח אותו לשרת — לחיפוש, לרישום, להתחברות ועוד. הטופס נפתח בתגית <form> ונסגר ב-</form>, ובתוכו יושבים הפקדים (inputs). לתגית <form> שני מאפיינים מרכזיים שקובעים לאן ו-איך נשלח המידע: 

<form action="http://www.dummy.com/form_details" method="post" autocomplete="on">
    <!-- Form fields will be here -->
</form>
  • action — הכתובת שאליה נשלח הטופס (בדרך כלל עמוד עם סקריפט בשרת). אם action ריק — המידע נשלח לאותו עמוד עצמו.
  • method — אופן השליחה, GET או POST (ראו למטה).
  • autocomplete — מאפיין מ-HTML5; אם דולק (on) הדפדפן זוכר ערכים ומציע אותם בפעם הבאה.

1.1 GET מול POST

שתי המתודות נבדלות בעיקר באיפה המידע נוסע — ולכן גם במה שרואים ובכמה אפשר לשלוח:

GETPOST
איפה נשלחבשורת הכתובת (URL) כ-querystringבגוף הבקשה, "מאחורי הקלעים" (לא ב-URL)
נראותגלוי בדפדפןלא נראה ב-URL
גודלמוגבל (~255 תווים)מידע רב / "אינסופי"
מתאים ל־מידע קטן, מהיר, לא רגישמידע גדול או מוסתר

מבנה ה-querystring שנוצר ב-GET: מתחיל ב-?, כל פקד הוא זוג name=value, והפרמטרים מופרדים ב-&. למשל חיפוש בגוגל: google.com/search?q=שנקר&ei=....

מלכודת מבחן: ב-GET המידע גלוי ב-URL — לכן אין לשלוח בו סיסמאות או מידע רגיש. גם שדה type="password" שנשלח ב-GET יופיע ב-querystring כטקסט גלוי.

מקור: 8.1_Lec_Forms (עמ' 4, 9–11)

2. פקדי קלט — <input> וסוגיו

לכל פקד יש name (שם — חובה) ו-value (ערך). השרת מקבל זוגות name=value. סוג הפקד נקבע במאפיין type. ברירת המחדל היא type="text". עיקרי הסוגים:

  • text — תיבת טקסט בת שורה אחת (ברירת המחדל).
  • email — אם הדפדפן תומך, מאמת כתובת דוא"ל תקנית בשליחה (דורש @); אם לא — מתנהג כ-text.
  • password — הטקסט מוצג מוצפן (נקודות); הערך עצמו נשלח כרגיל לשרת.
  • number — ספרות בלבד, עם חצי הגדלה/הקטנה, ותומך ב-min / max / step (גם ערכים לא שלמים כמו 5.8).
  • tel — מספרי טלפון; אינו כופה ספרות בלבד (רק מציג מקלדת ספרות בנייד).
  • checkbox — תיבת סימון (V). אם סומנה — נשלח ה-value; אם לא — לא נשלח כלום. קבוצה מוגדרת ע"י אותו name; value חובה; checked = מסומן כברירת מחדל.
  • radio — בחירה בודדת מקבוצה (עיגול). נשלח רק ה-value של הנבחר. הקבוצה מוגדרת ע"י אותו name; לא ניתן לבטל בחירה אלא ע"י בחירת כפתור אחר בקבוצה.
  • submit — לחצן ששולח את הטופס לפי מאפייני ה-<form>; ה-value הוא הכיתוב על הכפתור.
  • hidden — שדה סמוי; נשלח לשרת כרגיל (מופיע ב-querystring ב-GET), ולא מצמידים לו label.

(סוגים נוספים שנזכרו: url, color, search, range, date/time/month/week, file.) פקד type="file": הגולש אינו יכול לערוך את ה-value, וכדי להעלות בפועל את הקובץ לשרת יש לכתוב קוד צד-שרת. 

<label> Age:
    <input type="number" name="ageNum" value="25" min="20" max="120" step="5">
</label>

<h3>Hobbies:</h3>
<label><input type="checkbox" name="hobbies" value="Books"> Reading Books</label>
<label><input type="checkbox" name="hobbies" value="Sleeping"> Sleeping</label>

<input type="submit" value="Submit form">

הבחנה חוזרת במבחן: type="tel" אינו מגביל לספרות — type="number" כן. ובקבוצת checkbox/radio: פקד שלא סומן/נבחר לא שולח דבר (לא ערך ריק — פשוט חסר לגמרי מהבקשה).

מקור: 8.1_Lec_Forms (עמ' 16–33)

3. label, name מול id, וולידציה

3.1 התגית <label> וה-for

<label> מציין ליד כל פקד מה תפקידו. יש שתי דרכים לקשר תווית לפקד — עטיפה (הפקד בתוך ה-label) או קישור ב-for שמצביע על ה-id של הפקד: 

<!-- (1) עטיפה -->
<label>How old are you?
    <input name="age">
</label>

<!-- (2) קישור ב-for → id -->
<label for="age1">How old are you?</label>
<input name="age" id="age1">

name מול id — אל תבלבלו: name הוא השם שנשלח לשרת (מפתח ה-name=value) — חובה על כל פקד. id הוא מזהה בתוך העמוד — משמש את ה-label for ואת ה-JS/CSS, ואינו נשלח לשרת. אצל radio/checkbox כל הכפתורים בקבוצה חולקים name זהה, אך id חייב להיות ייחודי.

3.2 מאפייני ולידציה ועזרה

  • required — מחייב מילוי; שדה חובה ריק → הודעת שגיאה בשליחה.
  • pattern — (HTML5) regex לאימות הערך; הודעת השגיאה המותאמת מגיעה מ-title.
  • min / max — ערך מספרי/תאריך מינימלי ומקסימלי (בפקדי מספר, טווח ותאריך).
  • placeholder — טקסט עזרה שנעלם כשמקלידים; אינו נשלח לשרת.
  • maxlength — מגביל את מספר התווים שאפשר להקליד.
  • size — מספר התווים הגלויים בלבד — לא הגבלת קלט ולא רוחב.
<label> Coupon number:
    <input pattern="[a-c][0-6]" name="coupon"
           title="A coupon must be a-c followed by a digit 0-6">
</label>

שתי מלכודות קלאסיות: (1) size="6" אינו מגביל קלט — עם size=6 עדיין אפשר להקליד 20 תווים, פשוט רואים 6 בכל רגע; מי שמגביל הוא maxlength. (2) value הוא ערך ברירת מחדל שנשלח לשרת אם לא שונה — לעומת placeholder שהוא רק רמז ויזואלי שלעולם אינו נשלח.

מקור: 8.1_Lec_Forms (עמ' 12–15, 25)

4. select, textarea ושליחת הטופס

4.1 רשימה נגללת — <select> ו-<option>

<select> היא תיבת בחירה (drop-down); כל אפשרות היא <option>. ה-name יושב על <select> (שם הפרמטר שנשלח), וה-value יושב על כל <option> (האפשרות שנבחרה). selected קובע ברירת מחדל; size מציג מספר שורות גלויות; multiple מאפשר בחירה מרובה (עם Ctrl/Shift). 

<label>Number of Children:
    <select name="children">
      <option value="1">one child</option>
      <option value="3" selected>3 children</option>
      <option value="4">4 children</option>
    </select>
</label>

4.2 אזור טקסט — <textarea>

תיבה רב-שורתית. אין לה מאפיין value — התוכן יושב בין התגיות. cols = מספר הטורים (רוחב, ברירת מחדל 20); rows = מספר השורות (גובה, ברירת מחדל 1). 

<label> Address:
    <textarea name="Address" rows="3" cols="20"></textarea>
</label>

4.3 שליחה ומניעת ברירת המחדל

לחיצה על submit גורמת לדפדפן לשלוח את הטופס לכתובת ה-action ולנווט/לרענן. כשרוצים לטפל בטופס בצד הלקוח (למשל לשלוח עם fetch בלי לעזוב את העמוד) — מאזינים לאירוע submit וקוראים ל-preventDefault() כדי לבטל את הפעולה המובנית: 

const form = document.querySelector("form");
form.addEventListener("submit", function (e) {
  e.preventDefault();   // מבטל את השליחה/הניווט המובנה של הדפדפן
  // כאן: קוראים את הערכים ושולחים עם fetch, או מעבדים בצד לקוח
});

העשרה: מלבד ה-submit "האמיתי", אפשר לאמת גם ב-JS. אם מאזינים ל-submit ולא קוראים ל-preventDefault() — הדפדפן ימשיך לשלוח כרגיל. שימו לב גם שולידציית ה-HTML (required, pattern) חוסמת את השליחה עוד לפני שהאירוע נשלח — אלא אם מוסיפים novalidate ל-<form>. (חלק זה מעבר לשקפים, להעשרה.)

מקור: 8.1_Lec_Forms (עמ' 17, 28, 31–32) · preventDefault — העשרה

5. Bootstrap — מהו, חיבור והגריד

Bootstrap הוא Framework לעיצוב מהיר: אוסף מוכן של מחלקות CSS ורכיבים (components). הגרסה הנלמדת: v5.3.3 (getbootstrap.com). יתרונות: חוכמת המונים, עיצוב וגריד בלי מעצב, תכנות מהיר. חסרונות: עקומת למידה והרבה קוד מיותר שצריך לברור ממנו.

5.1 חיבור דרך CDN וסדר הקישורים

מוסיפים את Bootstrap דרך CDN (jsDelivr): <link> ל-CSS ו-<script> ל-JS:

<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css"
      rel="stylesheet">
<link rel="stylesheet" href="css/style.css">

<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js"></script>

סדר קריטי (מלכודת מבחן): ה-CSS של Bootstrap (ה-CDN) חייב לבוא לפני style.css שלכם — כי בשוויון ספציפיות ה-CSS שנטען אחרון מנצח, וכך הקוד שלכם דורס את Bootstrap.

5.2 Grid — מערכת ה-12 טורים

ה-grid רספונסיבי ומחולק ל-12 טורים. ההיררכיה: container ← row ← col. תא col ללא מספר מקבל רוחב שווה אוטומטי; תא col-8 תופס 8 טורים וכו'. סכום ה-col-* בשורה צריך להיות 12. 

<div class="container">
  <div class="row">
    <div class="col-8">רחב</div>
    <div class="col-4">צר</div>
  </div>
</div>

הגריד רספונסיבי לפי 6 נקודות שבירה (breakpoints):

<576px≥576px≥768px≥992px≥1200px≥1400px

6. רכיבים ומחלקות utility בסיסיות

Bootstrap מספק רכיבים מוכנים המורכבים ממחלקות. שלושת הרכיבים שנבנו בתרגול ("אתר Netflix"):

  • Navbar — nav.navbar.navbar-expand-lg ← container-fluid ← navbar-brand, כפתור המבורגר navbar-toggler (רספונסיבי), ו-ul.navbar-nav > li.nav-item > a.nav-link. הקלאס active רק על הקישור של העמוד הנוכחי.
  • Card — .card ← img.card-img-top + .card-body (בתוכו card-title, card-text וכפתור btn).
  • Modal — .modal ← .modal-dialog ← .modal-content (בתוכו modal-header / modal-body / modal-footer). נפתח בכפתור עם data-bs-toggle="modal" ו-data-bs-target="#id".

מחלקות utility בסיסיות: צבעי נושא לכפתורים — btn btn-primary / btn-secondary / btn-success / btn-danger / btn-warning / btn-info / btn-light / btn-dark / btn-link; רקעים — bg-dark, text-bg-dark (רקע כהה + טקסט בהיר); מצב כהה לרכיב — data-bs-theme="dark" (במקום .navbar-dark שהוצא משימוש ב-v5.3.0).

העשרה — איך יודעים איזה קלאס לדרוס? בתרגול הודגש: "לנחש זה בלתי אפשרי" — פותחים את devTools, מזהים את המחלקה של Bootstrap (למשל .navbar-brand), ודורסים אותה ב-style.css שלכם (בזכות סדר הקישורים הנכון). אם קלאס מוכן לא מספיק — מוסיפים CSS משלכם.

מקור: 8.4_Tar_Bootstrap (עמ' 7–26) · 8.2_Lec_Bootstrap (עמ' 3)
