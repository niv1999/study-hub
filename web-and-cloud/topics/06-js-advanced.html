1. פונקציות — הצהרה, ביטוי ופונקציית חץ

ב-JavaScript פונקציה היא גם ערך — אפשר לשמור אותה במשתנה, להעביר אותה כארגומנט, ולהחזיר אותה. יש שלוש דרכים נפוצות להגדיר פונקציה: 

// 1) Function Declaration — מוגדרת בשם
function add(x, y) {
    return x + y;
}

// 2) Function Expression — נשמרת במשתנה
const add2 = function (x, y) {
    return x + y;
};

// 3) Arrow Function (ES6)
const add3 = (x, y) => x + y;   // return מרומז כשאין סוגריים מסולסלים

כללי פונקציית החץ (arrow function): פרמטר יחיד → מותר בלי סוגריים עגולים; פקודה יחידה → מותר בלי סוגריים מסולסלים ואז ה-return מרומז; החץ => חייב לשבת באותה שורה של ההצהרה. ההבדל המהותי: לפונקציית חץ אין this משלה — ה-this הפנימי שווה לחיצוני (inner this = outer this).

1.1 פרמטרים דיפולטיביים

ערך דיפולטי נכנס לפעולה אך ורק בשני מצבים: כשלא הועבר ארגומנט, או כשהועבר undefined. ערכי false / NaN / null / 0 / "" הם ערכים תקינים ולכן לא מפעילים את הדיפולט: 

const doStuff = (value = "default") => console.log(value);

doStuff(undefined); // "default"  ← דיפולט
doStuff();          // "default"  ← דיפולט
doStuff(null);      // null
doStuff(false);     // false
doStuff(0);         // 0
doStuff("");        // "" (מחרוזת ריקה)

1.2 Hoisting — למה סדר ההצהרה קריטי

Hoisting הוא "הרמת" ההצהרות לראש הסקופ. משתני var מורמים ומאותחלים ל-undefined; משתני let/const מורמים אך לא מאותחלים — שימוש בהם לפני ההצהרה זורק שגיאה. לכן המרצה מבקש להצהיר תמיד לפני השימוש: 

console.log(a);   // undefined  (var הורם ואותחל ל-undefined)
var a = 3;

console.log(b);   // ReferenceError: Cannot access 'b' before initialization
let b = 4;

מלכודת מבחן קלאסית — var בתוך IIFE: 

var name = "Winnie-the-Pooh";
(function() {
    console.log(name);          // undefined  (!)
    var name = "Winnie-the-Pooh";
})();

ההצהרה הפנימית var name "מורמת" לראש הפונקציה, ולכן בשורת ה-console.log המשתנה המקומי כבר קיים אך עוד undefined — ולא מודפס הערך הגלובלי. זו בדיוק הסיבה שבקורס משתמשים ב-let/const בלבד.

העשרה — spread ו-rest (שניהם ...): spread "פותח" מערך/איטרבל (const b = [...a, 4], ואפילו [...'BC'] → ['B','C']); rest אוסף מספר לא-ידוע של ארגומנטים לתוך מערך וחייב להופיע אחרון ברשימת הפרמטרים (function f(a, b, ...rest)).

מקור: 6.1_Lec_JavaScript_Cont (עמ' 9–13, 54–60), 6.2_Tar_JavaScript_Functions

2. מערכים ומתודות

מערך הוא אוסף פריטים (אפשר מסוגים שונים) השמורים יחד בזיכרון. שני עקרונות: האורך דינמי — אין צורך לקבוע אותו באתחול; והאינדקס מתחיל ב-0. חשוב לזכור: typeof(arr) מחזיר 'object' — ולא 'array'. 

const nums = [1, 2, "more"];   // סוגים מעורבים
console.log(nums[0]);          // 1
console.log(nums.length);      // 3

nums.push("last");   // מוסיף לסוף        → 4 איברים
nums.pop();          // מסיר מהסוף
nums.shift();        // מסיר מההתחלה (אינדקס 0)
nums.unshift("x");   // מוסיף בהתחלה

מערך const — הבחנה שנבחנת: const נועל את ההשמה (הרפרנס), לא את התוכן. לכן מותר colors.push(...) למערך const, אבל colors = [...] (השמה מחדש) זורק TypeError: Assignment to constant variable.

העשרה — מתודות מסדר גבוה (higher-order): מעבר ל-push/pop/shift, כדאי להכיר את המתודות המובנות שמקבלות פונקציית callback (השקפים מפנים לרשימה המלאה ב-w3schools): 

const a = [1, 2, 3, 4];
a.forEach(n => console.log(n));      // רץ על כל איבר, לא מחזיר כלום
a.map(n => n * 2);                   // [2, 4, 6, 8]  — מערך חדש
a.filter(n => n % 2 === 0);          // [2, 4]        — מסנן
a.find(n => n > 2);                  // 3   — האיבר הראשון שמתאים
a.reduce((sum, n) => sum + n, 0);    // 10  — צובר לערך יחיד

מקור: 6.1_Lec_JavaScript_Cont (עמ' 6, 30–33), 6.2_Tar_JavaScript_Functions · מתודות מסדר גבוה — העשרה

3. אובייקטים ו-JSON

אובייקט הוא אוסף של זוגות key: value. יוצרים אותו כ-literal, וניגשים לשדה בשתי דרכים: נקודה (obj.width) או סוגריים מרובעים (obj["width"]). סוגריים מרובעים הכרחיים כשהמפתח נשמר במשתנה או מחושב דינמית: 

let rectWidth = 20;
let rect = { width: rectWidth, height: 100 };
console.log(rect.width);      // 20  (GET בנקודה)

const key = "name";
const obj = {};
obj[key] = "John Doe";        // SET בסוגריים — key דינמי
console.log(obj);             // { name: "John Doe" }

by-value מול by-reference (אין pointers ב-JS): משתנה פרימיטיבי (מספר, מחרוזת, בוליאני) מועבר לפונקציה כהעתק — שינוי בפונקציה אינו משפיע על המקור. לעומת זאת אובייקט או מערך מועברים by-reference — שינוי בתוך פונקציה אחרת כן משנה את המקור.

3.1 JSON — המרה בין מחרוזת לאובייקט

JSON (JavaScript Object Notation) הוא פורמט key-value לשליחת נתונים על גבי HTTP בין דפים/שרתים. שתי המתודות: JSON.parse ממחרוזת לאובייקט, ו-JSON.stringify מאובייקט למחרוזת. 

let jsonStr = '{"id":"123","action":"New file"}';
let jsonObj = JSON.parse(jsonStr);      // מחרוזת → אובייקט

console.log(jsonStr.action);   // undefined  ← נקודה על מחרוזת JSON!
console.log(jsonObj.action);   // "New file"
console.log(`${jsonObj}`);     // "[object Object]"  ← אובייקט בתוך template literal

let back = JSON.stringify(jsonObj);     // אובייקט → מחרוזת

שתי מלכודות מהשקף: (1) גישה עם נקודה למחרוזת JSON (jsonStr.action) מחזירה undefined — חייבים JSON.parse קודם. (2) הדפסת אובייקט בתוך template literal מציגה [object Object], לא את תוכנו.

מקור: 6.1_Lec_JavaScript_Cont (עמ' 34–37, 64–65), 6.1_Lec_JavaScript_Cont (עמ' 5)

4. ה-DOM — בחירה, קריאה ושינוי

ה-DOM (Document Object Model) הוא עץ שבו הדפדפן הופך כל תגית ל-node. ה-node הראשי הוא ה-root, וממנו מסתעפים צמתים-בנים (יחסי אב/בן/אחים כמו עץ משפחתי). טקסט בתוך תגית הוא text node בן של תגית האב.

4.1 בחירת אלמנטים

  • getElementById — מחזיר רפרנס בודד לאלמנט.
  • getElementsByTagName("a") / getElementsByClassName("x") — מחזירים אוסף (HTMLCollection) של רפרנסים.

הגדול מכולם: getElementsByTagName מחזיר אוסף, לא אלמנט יחיד — חייבים אינדקס [0] כדי לגשת לאלמנט: 

const main = document.getElementsByTagName("main")[0];   // [0] הכרחי!
const count = document.getElementsByTagName("img").length; // ספירת תמונות

4.2 קריאה, שינוי ויצירת תוכן

innerHTML קורא/כותב את ה-HTML הפנימי של אלמנט. הצבה ל-innerHTML דורסת את כל התוכן הקודם. דפוס נפוץ מהתרגול: בונים מחרוזת בלולאה ומציבים פעם אחת: 

const fruits = ["Banana", "Orange", "Apple"];
let list = "";
for (let i = 0; i < fruits.length; i++) {
    list += "<li>" + fruits[i] + "</li>";
}
document.getElementById("fruits-list").innerHTML = list;  // הצבה אחת

משנים עיצוב דרך אובייקט ה-style של האלמנט. שמות המאפיינים ב-camelCase (backgroundColor, fontSize) — לא כמו ב-CSS (background-color). ערך style.fontSize חוזר כמחרוזת עם יחידות ("20px"), אז חייבים parseInt לפני חישוב: 

document.getElementById("word").style.color = "blue";

let size = parseInt(p.style.fontSize);    // "20px" → 20
p.style.fontSize = size + 1 + "px";       // חייבים להחזיר "px"

העשרה — כלים מודרניים שלא בשקפים: querySelector("#id .cls") בוחר לפי סלקטור CSS (הראשון שמתאים), querySelectorAll את כולם. יצירת אלמנט: const li = document.createElement("li") ואז ul.appendChild(li). ניהול מחלקות: el.classList.add / remove / toggle("active") — נקי יותר מעריכת className ידנית.

מקור: 6.1_Lec_JavaScript_Cont (עמ' 38, 44–46), 6.2_Tar_JavaScript_Functions (עמ' 5, 13, 17, 21)

5. אירועים, scope וסגורים (closures)

event נוצר מהתרחשות (טריגר) — לחיצה, מעבר עכבר, טעינת עמוד — וקורא לקוד שמגיב. המרצה מבחין בין השיטה הישנה (חיבור inline ב-HTML, onclick="doIt()") שהיא ערבוב שכבות ולא רצויה, לבין השיטה החדשה: מחברים handler בקובץ JS חיצוני.

התחביר הנכון — בלי סוגריים! מציבים את שם הפונקציה כערך, לא קוראים לה: 

// property style
btn.onclick = changeTitle;               // ✔ ללא ()
btn.onclick = changeTitle();             // ✗ קורא מיד, ומציב את הערך המוחזר!

// addEventListener — התחביר המומלץ
btn.addEventListener("click", changeTitle);     // ✔ (שם האירוע, ואז ה-callback)
btn.addEventListener("click", changeTitle());   // ✗ הסוגריים מפעילים מיד

addEventListener("click", cb) — הארגומנט הראשון הוא שם האירוע כמחרוזת ("click", "mouseover"...), והשני הוא הפונקציה ללא סוגריים. סדר הפוך או סוגריים = באג.

בתוך handler שהוגדר כ-function רגילה, this מצביע על האלמנט שקיבל את האירוע — ולכן this.style.backgroundColor נותן את צבע אותו אלמנט. (לפונקציית חץ אין this משלה, לכן שם היא לא תתפוס את האלמנט.)

העשרה — event object ו-preventDefault: ה-callback מקבל אובייקט אירוע: addEventListener("submit", (e) => { e.preventDefault(); }). e.preventDefault() מבטל את פעולת ברירת המחדל של הדפדפן (למשל שליחת טופס או מעבר ב-<a>); e.target הוא האלמנט שהפעיל את האירוע.

5.1 סדר קריאת הדף — למה הסקריפט "לא רואה" את האלמנט

הדפדפן קורא מלמעלה למטה. סקריפט ב-<head> שניגש לאלמנט מה-<body> נכשל עם Cannot read properties of null, כי האלמנט עדיין לא נוצר. שני פתרונות: שים את הסקריפט בסוף ה-<body>, או עטוף ב-window.onload (נורה אחרי טעינת כל המשאבים): 

window.onload = () => {
    document.getElementById("btn").onclick = changeTitle;
};

5.2 Scope וסגורים (closures)

let/const הם block scoped; var הוא function scoped. closure הוא פונקציה ש"זוכרת" משתנים מהסקופ שבו נוצרה, גם אחרי שהסקופ הזה הסתיים. כך המרצה מטמיע תור פרטי — המערך collection "נסגר" בתוך המתודות ואינו נגיש מבחוץ: 

function Queue() {
    let collection = [];                       // פרטי — נסגר בתוך ה-closure
    this.enqueue = (el) => collection.push(el);
    this.dequeue = () => collection.shift();
    this.size    = () => collection.length;
}
let q = new Queue();
q.enqueue("first");
q.enqueue("second");
console.log(q.size());   // 2  — collection נשמר בין הקריאות
