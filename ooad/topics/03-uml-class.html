מהו UML ושתי משפחות הדיאגרמות

UML (Unified Modeling Language) היא שפה גרפית סטנדרטית לתיאור, ניתוח ותכן של מערכות תוכנה. הדיאגרמות מתחלקות לשתי משפחות:

  • דיאגרמות מבנה (structural): Class, Object, Package, Component, Deployment — מה קיים במערכת.
  • דיאגרמות התנהגות (behavioral): Use Case, Sequence, Communication, Activity, State — מה המערכת עושה ואיך.

מקור: UML.pdf

תרשים מחלקות — המבנה

תרשים מחלקות (Class Diagram) הוא ה«שרטוט» המבני של המערכת — המחלקות, התכונות, הפעולות והקשרים ביניהן. הדיאגרמה הנפוצה ביותר. כל מחלקה מיוצגת בקופסה בת שלושה תאים:

+-------------------------+
|        Circle           |   <- שם המחלקה
+-------------------------+
| - radius : float        |   <- תכונות (attributes)
| - center : Point        |
+-------------------------+
| + area() : double       |   <- פעולות (operations)
| + setRadius(r : float)  |
+-------------------------+

נראוּת (visibility): + public, - private, # protected.

מקור: Exercise4-ClassDiagrams.pdf

קשרים בין מחלקות

חמשת הקשרים המרכזיים (חשוב לזהות את הסימון!):

  • הורשה / הכללה (Generalization): חץ עם משולש ריק △ מהתת-מחלקה אל העל-מחלקה («is-a»).
  • אסוציאציה (Association): קו רגיל — → קשר מבני בין מחלקות («knows-a»).
  • צבירה (Aggregation): מעוין ריק ◇— — «has-a» חלש; החלק יכול להתקיים בלי השלם.
  • הרכבה (Composition): מעוין מלא ◆— — «has-a» חזק; החלק נהרס יחד עם השלם.
  • תלות (Dependency): חץ מקווקו --▶ — שימוש זמני («uses»).

Aggregation מול Composition: ההבדל הוא במשך החיים — בהרכבה (◆) החלק «שייך בלעדית» ונהרס עם השלם (למשל חדרים בבית); בצבירה (◇) החלק עצמאי (למשל שחקנים בקבוצה).

ריבוי (Multiplicity)

על קצות האסוציאציה מציינים ריבוי: כמה מופעים משתתפים בקשר.

  • 1 — בדיוק אחד. 0..1 — אפס או אחד (אופציונלי).
  • * או 0..* — אפס או יותר. 1..* — אחד או יותר.

למשל: Order 1 —— * LineItem — הזמנה אחת מכילה מספר שורות.
