תבניות התנהגות — תקשורת

תבניות התנהגות (Behavioral) עוסקות באופן התקשורת וחלוקת האחריות בין אובייקטים — אלגוריתמים והקצאת תפקידים. בקורס: Observer, Template Method, Strategy, Iterator, Command.

מקור: Behavioral-Design-Patterns.pdf

Observer

מטרה: תלות אחד-לרבים — כשאובייקט (Subject) משנה מצב, כל התלויים בו (Observers) מקבלים הודעה ומתעדכנים אוטומטית.

class Observer { virtual void update() = 0; };
class Subject {
  vector<Observer*> observers;
  void notify() { for (auto o : observers) o->update(); }
};

שימוש: מערכות אירועים, התראות MVC, pub/sub.

מקור: Behavioral-Design-Patterns.pdf

Strategy ו-Template Method

Strategy: להגדיר משפחת אלגוריתמים, לעטוף כל אחד, ולהפוך אותם לניתנים להחלפה בזמן ריצה. הלקוח בוחר אסטרטגיה דרך ממשק משותף. דוגמה: אלגוריתמי מיון, שיטות תשלום, אסטרטגיות דחיסה.

Template Method: להגדיר את שלד האלגוריתם במחלקת הבסיס, ולהשאיר לתת-מחלקות למלא שלבים ספציפיים. דוגמה: frameworks, לולאות משחק, צינורות ייבוא נתונים.

Strategy מול Template Method: Strategy מחליף אלגוריתם שלם דרך הרכבה (composition) בזמן ריצה; Template Method משנה שלבים דרך הורשה (inheritance) בזמן קומפילציה.

מקור: Exercise12-DesignPatternsStrategyTemplateIterator.pdf

Iterator ו-Command

Iterator: לספק דרך אחידה לעבור על אוסף ברצף, בלי לחשוף את הייצוג הפנימי שלו. דוגמה: איטרטורים של STL, לולאות range-based, cursors של מסד נתונים.

Command: לעטוף בקשה כאובייקט — מאפשר לפרמטר בקשות, לתייר/לתעד אותן, ולתמוך בפעולות ניתנות לביטול (undo/redo). דוגמה: פעולות GUI (undo/redo), תורי משימות, הקלטת מאקרו.

מקור: Behavioral-Design-Patterns.pdf
