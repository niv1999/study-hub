מצביעים חכמים (Smart Pointers)

מקור: Smart-Pointers.pdfמקור: OOAD_Course_Summary.pdf

למה מצביעים חכמים?

מצביע חכם (smart pointer) הוא אובייקט העוטף מצביע גולמי (raw pointer) ומנהל את משך החיים של המשאב אוטומטית — בלי delete ידני. הוא מבוסס על עקרון RAII (Resource Acquisition Is Initialization): המשאב משוחרר כשהאובייקט העוטף יוצא מ-scope. כך נמנעות דליפות זיכרון (memory leaks) ומצביעים תלויים (dangling pointers).

ב-C++ המודרני (מ-C++11) מעדיפים מצביעים חכמים על פני new/delete ידני. שלושת הסוגים ב-<memory>: unique_ptr, shared_ptr, weak_ptr.

מקור: Smart-Pointers.pdf

unique_ptr — בעלות בלעדית

בעלות בלעדית (exclusive ownership): בעלים יחיד של המשאב. לא ניתן להעתיק — רק להעביר (move). משמיד את האובייקט אוטומטית ביציאה מ-scope. אפס תקורה (zero overhead) לעומת מצביע גולמי.

std::unique_ptr<Widget> w = std::make_unique<Widget>();
auto w2 = std::move(w);   // העברת בעלות; w עכשיו null
// w2 משוחרר אוטומטית בסוף ה-scope

ברירת המחדל כשהבעלות יחידה וברורה. השתמשו ב-make_unique ליצירה.

מקור: Smart-Pointers.pdf

shared_ptr — בעלות משותפת

בעלות משותפת (shared ownership): כמה בעלים חולקים את אותו משאב באמצעות ספירת הפניות (reference counting). האובייקט מושמד כאשר ה-shared_ptr האחרון משוחרר (הספירה יורדת ל-0). יש תקורה קטנה מעדכוני מונה אטומיים.

auto a = std::make_shared<Node>();
auto b = a;              // ספירה = 2
b.reset();               // ספירה = 1
// המשאב ישוחרר כשגם a ייהרס (ספירה = 0)

שימוש: כשכמה אובייקטים חייבים להחזיק את אותו משאב יחד.

מקור: Smart-Pointers.pdf

weak_ptr — צופה ללא בעלות

צופה ללא בעלות (non-owning observer): מתבונן ב-shared_ptr בלי להעלות את ספירת ההפניות. חייבים לנעול אותו (lock() → shared_ptr) לפני שימוש, מה שגם בודק בבטחה אם האובייקט עדיין קיים.

std::weak_ptr<Node> wp = a;         // לא מעלה ספירה
if (auto sp = wp.lock()) {          // sp הוא shared_ptr תקף
    sp->use();
} // else: האובייקט כבר נהרס

התפקיד המרכזי של weak_ptr: לשבור הפניות מעגליות (circular references) שגורמות לדליפת זיכרון — וכן ל-caches ו-back-pointers.
