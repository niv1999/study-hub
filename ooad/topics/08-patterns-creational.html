מהי תבנית עיצוב?

תבנית עיצוב (Design Pattern) היא פתרון כללי, מנוסה ורב-שימושי לבעיית עיצוב חוזרת. מקור: ספר «Gang of Four» (Gamma, Helm, Johnson, Vlissides, 1995), 23 תבניות בשלוש משפחות. תבניות יצירה (Creational) עוסקות באופן יצירת האובייקטים — הפרדת הלקוח מפרטי היצירה.

מקור: Design-Patterns-Creational.pdf

Singleton

מטרה: להבטיח שמופע יחיד של מחלקה קיים, ולספק אליו גישה גלובלית.

class Config {
public:
  static Config& instance() {
    static Config c;   // נוצר פעם אחת, thread-safe מ-C++11
    return c;
  }
  Config(const Config&) = delete;   // אין העתקה
private:
  Config() {}
};

שימוש: Config manager, Logger, Thread pool — משאב יחיד משותף.

מקור: Design-Patterns-Creational.pdf

Prototype

מטרה: ליצור אובייקטים חדשים על-ידי שכפול (clone) של אובייקט קיים, במקום בנייה מאפס. שימושי כשהאתחול יקר, או כדי להימנע מתת-מחלקות לכל וריאציה.

class Shape {
public:
  virtual Shape* clone() const = 0;  // מתודת השכפול
};
class Circle : public Shape {
  Shape* clone() const override { return new Circle(*this); }
};

מקור: Exercise8-DesignPatternsSingletonPrototype.pdf

Abstract Factory ו-Builder

Abstract Factory: ממשק ליצירת משפחות של אובייקטים קשורים בלי לציין מחלקות קונקרטיות. דוגמה: ערכת UI חוצת-פלטפורמות שיוצרת Button ו-Checkbox מתאימים לכל מערכת הפעלה.

Builder: מפריד את הבנייה שלב-אחר-שלב של אובייקט מורכב מהייצוג הסופי שלו — כך אותו תהליך בנייה יכול לייצר ייצוגים שונים. דוגמה: מחולל מסמכים, בונה שאילתות, אובייקטי הגדרה מורכבים.

Abstract Factory יוצרת משפחת מוצרים בבת-אחת; Builder בונה מוצר מורכב יחיד בשלבים.

מקור: Design-Patterns-Creational.pdf
