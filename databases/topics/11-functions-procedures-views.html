Views

View הוא שאילתה שמורה שמתנהגת כטבלה וירטואלית — אינו מאחסן נתונים אלא מריץ את השאילתה בכל פנייה. שימושי להפשטה, אבטחה (חשיפת עמודות מסוימות) ושימוש חוזר:

CREATE VIEW high_earners AS
SELECT first_name, salary FROM employee
WHERE salary > 10000;

SELECT * FROM high_earners;   -- כמו טבלה

Stored Functions

פונקציה מאוחסנת מקבלת פרמטרים ומחזירה ערך יחיד (RETURNS), ושמישה בתוך שאילתות:

CREATE FUNCTION annual_salary(m DECIMAL(10,2))
RETURNS DECIMAL(10,2)
DETERMINISTIC
RETURN m * 12;

SELECT first_name, annual_salary(salary) FROM employee;

Stored Procedures

פרוצדורה היא בלוק פקודות שמורות שמופעל עם CALL. יכולה לקבל פרמטרים IN (קלט), OUT (פלט) ו-INOUT, ואינה חייבת להחזיר ערך:

CREATE PROCEDURE emp_count(IN dept INT, OUT cnt INT)
BEGIN
  SELECT COUNT(*) INTO cnt FROM employee WHERE dept_id = dept;
END;

CALL emp_count(10, @c);  SELECT @c;

פונקציה מול פרוצדורה

  • פונקציה: מחזירה ערך יחיד, נקראת בתוך SELECT, בלי פרמטרי OUT.
  • פרוצדורה: מופעלת ב-CALL, יכולה לבצע כמה פעולות ולהחזיר דרך OUT, לא בתוך שאילתה.

🧪 מעבדת SQL — תרגול חי

כתבו SQL אמיתי והריצו אותו כאן, ישירות בדפדפן. ▶ הרצה מציגה את התוצאה שלכם; בדיקה מול פתרון משווה אותה לתוצאת הפתרון הרשמי.

המנוע (SQLite ב-WebAssembly) רץ כולו בדפדפן ותואם ל-MySQL בכל מה שנלמד בקורס — AUTO_INCREMENT, DESCRIBE ו-TRUNCATE מתורגמים אוטומטית. כל הרצה מתחילה ממסד נקי, אז אי-אפשר לקלקל כלום.

🧪 משימה: צרו VIEW בשם high_earners עם שם ושכר של העובדים שמרוויחים מעל 8000 — ואז שלפו ממנו את כל השורות.

CREATE VIEW high_earners AS
SELECT first_name, salary FROM employee
WHERE salary > 8000;

SELECT * FROM high_earners;

🧪 משימה: סנדבוקס חופשי — נסו כאן כל דבר על טבלת employee. הערה: פונקציות ופרוצדורות מאוחסנות אינן נתמכות במנוע הדפדפן (SQLite) — אותן מתרגלים ב-MySQL אמיתי; Views נתמכים במלואם.
