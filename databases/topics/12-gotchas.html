NULL — המלכודת הכי נבחנת

  • = NULL / <> NULL תמיד מחזירים UNKNOWN — לעולם לא TRUE. תמיד IS NULL / IS NOT NULL.
  • COUNT(col) מדלג על NULL; COUNT(*) סופר את כל השורות.
  • AVG/SUM מתעלמים מ-NULL (הממוצע על ערכים לא-NULL בלבד).
  • NOT IN (…, NULL) לעולם אינו TRUE — מלכודת שמבליעה שורות. עדיף LEFT JOIN … WHERE … IS NULL, או לוודא שאין NULL ברשימה.
  • NULL «עובר» אילוץ CHECK (UNKNOWN ≠ FALSE) — צריך גם NOT NULL.

מקור: constraints_and_triggers.pdf

WHERE מול HAVING, וסדר הביצוע

WHERE מסנן שורות לפני הקיבוץ ואינו יכול להשתמש באגרגציה; HAVING מסנן קבוצות אחרי GROUP BY. סדר הביצוע הלוגי: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT — ולכן כינוי (alias) מ-SELECT לרוב לא זמין ב-WHERE.

מקור: sql_joins_agg.pdf

GROUP BY — עמודות לא-מקובצות

כל עמודה ב-SELECT שאינה בתוך אגרגציה חייבת להיות ב-GROUP BY. אחרת, במצב ONLY_FULL_GROUP_BY מתקבלת שגיאה; ב-MySQL ישן היה מוחזר ערך «אקראי» מהקבוצה — מקור לבאגים שקטים.

מקור: sql_joins_agg.pdf

JOIN — מכפלות ו-OUTER

  • שכחת ON (או פסיק בין טבלאות בלי תנאי) → מכפלה קרטזית ענקית.
  • תנאי על הטבלה החיצונית ב-WHERE (במקום ב-ON) הופך LEFT JOIN בפועל ל-INNER.
  • להחליף INNER ב-LEFT יכול לשנות ספירות (נשמרות שורות ללא התאמה עם NULL).

מקור: sql_intro_joins.pdf

DML מסוכן ואילוצים

  • UPDATE/DELETE ללא WHERE משנים/מוחקים את כל הטבלה.
  • DROP מוחק את הטבלה; DELETE מוחק שורות; TRUNCATE מרוקן ומאפס מונה.
  • הכנסת ערך מפתח-זר שאינו קיים ב-אב → שגיאת הפרת FK.
  • DECIMAL לכסף (לא FLOAT — שגיאות עיגול).

מקור: ddl_insert_update_delete.pdf

LIKE, DISTINCT ו-View

  • התו הכללי ב-LIKE הוא % (ו-_ לתו יחיד) — לא *.
  • DISTINCT מסיר כפילויות מהתוצאה בלבד; אינו ממיין ואינו משנה נתונים.
  • View אינו מאחסן נתונים — הוא מריץ את שאילתת הבסיס בכל פנייה.
  • פונקציה מחזירה ערך ונקראת ב-SELECT; פרוצדורה מופעלת ב-CALL.
