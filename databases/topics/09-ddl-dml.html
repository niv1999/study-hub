DDL — הגדרת מבנה

DDL (Data Definition Language) מגדיר את מבנה המסד:

CREATE TABLE employee (
  emp_id     INT PRIMARY KEY,
  first_name VARCHAR(50) NOT NULL,
  salary     DECIMAL(10,2),
  dept_id    INT,
  FOREIGN KEY (dept_id) REFERENCES department(dept_id)
);

ALTER TABLE employee ADD COLUMN email VARCHAR(100);
DROP TABLE employee;   -- מוחק את הטבלה כולה

מקור: ddl.sql

טיפוסי נתונים נפוצים

  • INT / BIGINT — מספרים שלמים.
  • DECIMAL(p,s) — מספר עשרוני מדויק (כסף). FLOAT/DOUBLE — קירוב.
  • VARCHAR(n) — טקסט משתנה עד n. CHAR(n) — קבוע. TEXT — ארוך.
  • DATE, DATETIME, TIMESTAMP, BOOLEAN.

DML — שינוי נתונים

INSERT INTO employee (emp_id, first_name, salary)
VALUES (1, 'Dana', 8000);

UPDATE employee SET salary = salary * 1.1
WHERE dept_id = 10;

DELETE FROM employee WHERE emp_id = 1;

מסוכן: UPDATE/DELETE ללא WHERE משנים/מוחקים את כל השורות. תמיד לוודא WHERE.

מקור: ddl_insert_update_delete.pdf

DROP מול DELETE מול TRUNCATE

  • DELETE FROM t; — DML, מוחק שורות (עם WHERE אופציונלי), ניתן לביטול בטרנזקציה.
  • TRUNCATE TABLE t; — DDL, מרוקן את כל הטבלה מהר, מאפס AUTO_INCREMENT, לרוב לא ניתן לביטול.
  • DROP TABLE t; — DDL, מוחק את הטבלה עצמה (מבנה + נתונים).
