מודול ↗

3NFעיצוב ונירמול

צורה נורמלית שלישית — 2NF ואין תלות מעבר (עמודה לא-מפתח תלויה בעמודה לא-מפתח).

מודול ↗

Aggregate FunctionsSQL

COUNT/SUM/AVG/MIN/MAX — מכווצות קבוצת שורות לערך יחיד.

מודול ↗

ALTER TABLESQL

משנה מבנה טבלה קיימת: ADD / DROP / MODIFY עמודה.

מודול ↗

Attribute (עמודה)יסודות ומודל

עמודה בטבלה, בעלת שם וטיפוס (domain).

מודול ↗

AUTO_INCREMENTSQL

נותן למפתח ערך רץ אוטומטי (1,2,3…) בכל הוספה.

מודול ↗

BCNFעיצוב ונירמול

Boyce-Codd — חיזוק 3NF: לכל תלות X→Y, ה-X הוא מפתח-על.

מודול ↗

Candidate Keyיסודות ומודל

מפתח מועמד — מפתח-על מינימלי (אין תת-קבוצה שלו שהיא מפתח).

מודול ↗

Cardinalityעיצוב ונירמול

קרדינליות — כמה ישויות מכל צד משתתפות בקשר (1:1, 1:N, N:M).

מודול ↗

CHECKאילוצים ואובייקטים

אילוץ שכל שורה חייבת לקיים תנאי; NULL «עובר» את הבדיקה.

מודול ↗

Constraintאילוצים ואובייקטים

אילוץ — כלל שהמסד אוכף (PRIMARY/FOREIGN KEY, UNIQUE, CHECK, NOT NULL, DEFAULT).

מודול ↗

CREATE TABLESQL

יוצר טבלה עם עמודות, טיפוסים ואילוצים; אפשר גם CREATE TABLE t AS SELECT.

מודול ↗

CROSS JOINSQL

מכפלה קרטזית — כל שילוב שורות, ללא תנאי.

מודול ↗

DBMSיסודות ומודל

מערכת ניהול מסד נתונים — התוכנה שמנהלת אחסון, שליפה, גישה ושלמות (למשל MySQL).

מודול ↗

DDLSQL

Data Definition Language — CREATE/ALTER/DROP להגדרת מבנה.

מודול ↗

DESCRIBESQL

מציג את מבנה הטבלה — שם עמודה, טיפוס, Null, Key, Default.

מודול ↗

DISTINCTSQL

מסיר שורות כפולות מהתוצאה.

מודול ↗

DMLSQL

Data Manipulation Language — INSERT/UPDATE/DELETE לשינוי נתונים.

מודול ↗

Entity (ישות)עיצוב ונירמול

«דבר» בעולם המיוצג בטבלה (עובד, מחלקה).

מודול ↗

Entity Integrityיסודות ומודל

שלמות ישויות — מפתח ראשי לעולם אינו NULL.

מודול ↗

ER Modelעיצוב ונירמול

מודל ישות-קשר — עיצוב רעיוני עם ישויות, תכונות וקשרים.

מודול ↗

Foreign Keyאילוצים ואובייקטים

מפתח זר — עמודה שמצביעה על מפתח ראשי בטבלה אחרת ויוצרת קשר; אוכפת שלמות התייחסות.

מודול ↗

Functional Dependencyעיצוב ונירמול

תלות פונקציונלית X→Y — ערך X קובע חד-משמעית את Y.

מודול ↗

GROUP BYSQL

מקבץ שורות לפי ערך ומפעיל אגרגציה לכל קבוצה.

מודול ↗

HAVINGSQL

מסנן קבוצות אחרי GROUP BY; יכול להשתמש באגרגציה.

מודול ↗

Indexיסודות ומודל

קובץ אינדקס לזירוז חיפוש על עמודה; מפתח ראשי מקבל אינדקס אוטומטית.

מודול ↗

INNER JOINSQL

מחזיר רק זוגות תואמים משתי הטבלאות.

מודול ↗

IS NULLSQL

הדרך הנכונה לבדוק ערך חסר; = NULL תמיד UNKNOWN.

מודול ↗

JOINSQL

חיבור שורות מטבלאות לפי תנאי (בדרך כלל מפתח-זר = מפתח-ראשי).

מודול ↗

Junction Table (טבלת קשר)עיצוב ונירמול

טבלה הממפה קשר N:M עם שני מפתחות זרים; הצירוף הוא המפתח הראשי.

מודול ↗

LEFT JOINSQL

כל שורות הטבלה השמאלית + התאמות; ללא התאמה → NULL בצד ימין.

מודול ↗

LIKESQL

התאמת תבנית טקסט: % (רצף כלשהו), _ (תו יחיד).

מודול ↗

LIMITSQL

מגביל את מספר השורות המוחזרות (עם OFFSET לדילוג).

מודול ↗

Normalizationעיצוב ונירמול

נירמול — פירוק טבלאות להסרת כפילות ואנומליות.

מודול ↗

ON DELETE CASCADEאילוצים ואובייקטים

מחיקת שורת אב מוחקת אוטומטית את השורות התלויות בה.

מודול ↗

ORDER BYSQL

ממיין את התוצאה לפי עמודות (ASC/DESC).

מודול ↗

Primary Keyיסודות ומודל

מפתח ראשי — עמודה/צירוף שמזהה כל שורה ביחידות; ייחודי ולא-NULL, אחד לטבלה.

מודול ↗

Referential Integrityאילוצים ואובייקטים

שלמות התייחסות — כל ערך מפתח זר תואם למפתח ראשי קיים או NULL.

מודול ↗

Relation (טבלה)יסודות ומודל

טבלה במודל הרלציוני; אוסף שורות (tuples) עם עמודות (attributes).

מודול ↗

Relationship (קשר)עיצוב ונירמול

חיבור בין ישויות ב-ER; מיוצג במעוין.

מודול ↗

Schemaיסודות ומודל

הסכימה — מבנה המסד (טבלאות, עמודות, טיפוסים, אילוצים), להבדיל מהמופע (הנתונים).

מודול ↗

SELECTSQL

פקודת השליפה: בוחרת עמודות משורות המקיימות תנאי.

מודול ↗

Self-joinSQL

חיבור טבלה לעצמה (שני כינויים) — לקשרים היררכיים כמו עובד↔מנהל.

מודול ↗

SQLSQL

Structured Query Language — שפת התקן לשליפה ולעדכון של מסדי נתונים רלציוניים.

מודול ↗

Stored Functionאילוצים ואובייקטים

פונקציה מאוחסנת שמחזירה ערך יחיד ונקראת בתוך שאילתה.

מודול ↗

Stored Procedureאילוצים ואובייקטים

בלוק פקודות שמורות שמופעל ב-CALL; פרמטרי IN/OUT/INOUT.

מודול ↗

SubquerySQL

שאילתה בתוך שאילתה (ב-WHERE/FROM/SELECT).

מודול ↗

Super Keyיסודות ומודל

מפתח-על — קבוצת עמודות שמזהה שורה ביחידות (לאו דווקא מינימלית).

מודול ↗

Transitive Dependencyעיצוב ונירמול

תלות מעבר — A→B ו-B→C, כך ש-C תלוי ב-A דרך B; מפרה 3NF.

מודול ↗

Triggerאילוצים ואובייקטים

קוד שרץ אוטומטית BEFORE/AFTER על INSERT/UPDATE/DELETE; גישה דרך NEW/OLD.

מודול ↗

TRUNCATESQL

מרוקן טבלה מהר (DDL), מאפס AUTO_INCREMENT, לרוב ללא rollback.

מודול ↗

Tuple (שורה)יסודות ומודל

שורה בטבלה — ישות בודדת שערכיה מתאימים לעמודות.

מודול ↗

UNIQUEאילוצים ואובייקטים

אילוץ ייחודיות; מתיר NULL (בשונה ממפתח ראשי).

מודול ↗

Update Anomalyעיצוב ונירמול

אנומליית עדכון — עובדה כפולה שעודכנה חלקית ויוצרת חוסר-עקביות.

מודול ↗

VARCHAR / DECIMALSQL

VARCHAR(n) טקסט משתנה; DECIMAL(p,s) מספר עשרוני מדויק (מתאים לכסף).

מודול ↗

Viewאילוצים ואובייקטים

שאילתה שמורה כטבלה וירטואלית; אינה מאחסנת נתונים.

מודול ↗

Weak Entityעיצוב ונירמול

ישות חלשה — אינה מזדהה לבד; תלויה בישות חזקה לזיהוי.

מודול ↗

WHERESQL

פסקה המסננת שורות לפני קיבוץ; אינה יכולה להשתמש באגרגציה.
